<h1>Les espaces vectoriels</h1>



<h2>Un type précis de structure</h2>

<h3>Qu'est ce qu'est un espace vectoriel ?</h3>
<p>
Grâce aux corps, nous pouvons définir une nouvelle structure algébrique particulièrement utile : l'espace vectoriel.
<span class="important"><definition name="vector_space"></span>.
Bien que ce concept puisse sembler complexe, il est la base de choses très importantes et basiques en mathématiques.
Prenons un corps "K" sur un ensemble "A", et un espace vectoriel "E" sur un ensemble "B".
</p>
<math>K = (A, +, .)</math>
<p>
Dans ce cas, l'espace vectoriel "E" admet une addition, tel que cette addition forme un groupe abélien sur "B".
</p>
<math>(B, +)</math>
<p>
En suite, il existe une loi externe à gauche entre un élément de "A" et un élément de "B" vers un élément de "B".
Généralement, cette loi est notée comme une multiplication, avec généralement un "*".
</p>
<math><mforall>a <min> A, <mforall>b <min> B, <mexists> c <min> B, a * b = c</math>
<p>
Attention : <span class="important">il est impossible d'utiliser cette loi entre deux éléments de "B", seulement entre un élément de "A" et un élément de "B"</span>.
Cependant, il est possible d'utiliser cette loi sur une opération entre des élements de "A" qui donnent un nouvel élément de "A", avec une opérations entre des éléments de "B", qui donnent un nouvel élément de "B".
Cette propriété va nous être très utile pour les propriétés qui suivront.
Déjà, <span class="important">cette loi est associative, ce qui veut dire qu'opérer deux éléments "a" et "c" de "A" avec la deuxième opération du corps puis utiliser cette opération avec un élément "b" de "B" est égal à opérer "c" et "b", puis "a" au résultat</span>.
Si l'on considère la deuxième opération du corps comme une multiplication et la loi externe de l'espace vectoriel comme une multiplication aussi, alors cela revient à dire que l'ordre de réalisation des multiplications n'importent pas.
</p>
<math><mforall>a, c <min> A, <mforall>b <min> B, (a * c) * b = a * (c * b)</math>
<p>
La distributivité à gauche s'écrit comme dans un anneau, à la différence prés qu'il faut faire attention à l'ordre des opérandes pour respecter l'opération :
</p>
<math><mforall>a <min> A, <mforall>b, d <min> B, a * (b + d) = a * b + a * d</math>
<p>
C'est la même chose pour la distributivité à droite.
</p>
<math><mforall>a, c <min> A, <mforall>b <min> B, (a + c) * b = a * b + c * b</math>
<p>
Finalement, l'élément neutre du corps pour sa deuxième opération est aussi un élément neutre pour cette opération...
</p>
<math><mforall>b <min> B, <mexists>e <min> A, b * e = b</math>
<p>
Si vous avez toutes ces conditions grâce à un corps "K", vous avec un espace vectoriel sur "K".
L'exemple le plus simple d'espace vectoriel sur "K" est... "K".
En effet, <span class="important">"K" est un espace vectoriel sur lui même</span>.
Comme "K" est un corps, sa première loi est bel et bien un groupe abélien, et sa deuxième loi se comporte comme une opération d'un élément de l'ensemble de "K" et... de l'ensemble de "K" (avec associativité, neutre et distributivité).
</p>
<p>
Plaçons nous dans le corps le plus simple pour définir des espaces vectoriels : "R".
Déjà, comme nous l'avons vue, "R" est un espace vectoriel sur lui même.
Or, <span class="important">il est possible de construire des ensembles de tous les "n-uplets" de "R", qui sont eux aussi des espaces vectoriels</span> (généralement, nous noterons ces ensembles "R exposant n", comme avec un produit cartésien).
Il faut cependant bien définir leurs propriétés.
Déjà, <span class="important">nous allons définir une opération d'addition entre les "n-uplets", où la somme de deux "n-uplets" donne un nouveau "n-uplet" ou le k-ième élément du résultat représente la somme du k-ième élément du premier "n-uplet" et du k-ième élément du second "n-uplet"</span>.
Comme cette opération se comporte comme plusieurs additions dans "R" disjointes, alors la structure de ces "n-uplets" et de cette opération ont les mêmes propriétés que dans "R" : un groupe abélien par cette addition.
</p>
<math><mforall> (a, b, ..., n), (a', b', ..., n') <min> <mi>R</mi><msup>n</msup>, (a, b, ..., n) + (a', b', ..., n') = (a + a', b + b', ..., n + n')</math>
<p>
En suite, <span class="important">nous allons définir une opération entre ces "n-uplets" et "R", où l'on multiplie chaque élément du "n-uplet" par un élément de "R"</span>.
Comme cette opération se comporte comme plusieurs multiplications dans "R" disjointes, alors la structure de ces "n-uplets" et de cette opération ont les mêmes propriétés que dans "R" : associativité, neutre (1) et distributivité.
</p>
<math><mforall> (a, b, ..., n) <min> <mi>R</mi><msup>n</msup>, <mforall> x <min> R, x * (a, b, ..., n) = (a * x, b * x, ..., n * x)</math>
<p>
Donc, <span class="important">"R exposant n" est un espace vectoriel sur "R"</span>.
C'est d'ailleurs grâce à ce résultat que l'on peut démontrer que "R exposant 2" et "R exposant 3" sont des espaces vectoriels sur "R".
Grâce à ce résultat, <span class="important">on peut lier géométrie et algèbre, puisque le plan 2D et l'espace 3D sont tout deux des espaces vectoriels</span>.
Ce résultat nous donne une façon précise de définir certains types précis de transformation géométrique, grâce à de l'algèbre.
</p>
<p>
Comme pour beaucoup de structures algébriques de ce genre, si il existe un "espace vectoriel", il existe la notion de "sous-espace vectoriel".
<span class="important"><definition name="sub_vector_space"></span>.
Un exemple très simple : l'ensemble des vecteurs de la forme "(a, 0)" avec "a" un nombre réel quelconque de "R²" forme un sous-espace vectoriel de "R²".
En effet, additionner deux éléments de ce genre "(b, 0)" et "(c, 0)" revient à calculer "(b + c, 0)" (qui appartient toujours à cet espace), et multiplier "(d, 0)" par un réel "e" revient à calculer "(e * d, 0)" (qui appartient toujours à cet espace).
Graphiquement parlant, ce sous-espace vectoriel de "R²" représente... la droite des abscisses.
</p>
<graphic><background_color white><line name=type_number() x_1=-10 y_1=0 x_2=10 y_2=0></graphic>
<p>
Avec ce concept, on peut en introduire un autre qu'il lui est directement lié : les vecteurs.
<span class="important"><definition name="vector_space-vector"></span>.
Par exemple, dans l'espace vectoriel formé par "R²" (comme défini plus haut), "(4, 8)" est un vecteur.
</p>
<p>
Les espaces vectoriels possèdent aussi des propriétés permettant de définir un type de formule précis : les combinaisons linéaires.
<span class="important"><definition name="linear_combination"></span>.
Le nom "linéaire" provient du fait qu'une combinaison linéaire est une somme de termes représentant des fonctions linéaires (un scalaire multiplié par un vecteur de l'espace).
En fait, une combinaison linéaire dans un espace vectoriel représente une valeur qui est toujours dans l'espace vectoriel, puisque le produit d'éléments de "K" et de vecteurs donne un vecteur, et la somme de vecteurs donne un vecteur.
Grâce à ça, <span class="important">on peut utiliser des indéterminées facilement dans un espace vectoriel, en représentant des éléments de "K" indéterminées, ou même des vecteurs indéterminées, permettant de résoudre des équations</span>.
Voici quelques exemples de combinaisons linéaires possibles, dans "R²".
</p>
<math>4 * (7, 8) + 2 * (-4, 1) = u <mequi> u = (4 * 7 + 2 * -4, 4 * 8 + 2 * 1) = (20, 34)</math></br>
<math>-2 * (1, 8) + -a * (1, 4) = 4 * (1, 2) <mequi> -2 * (1, 8) + -4 * (1, 2) = a * (1, 4) <mequi> (-2, -16) + (-4, -8) = a * (1, 4) <mequi> (-6, -24) = a * (1, 4) <mequi> a = -6</math></br>
<math><mi>a</mi><msub>0</msub>, <mi>a</mi><msub>1</msub>, ..., <mi>a</mi><msub>n</msub> <min> R, <mi>w</mi><msub>0</msub>, <mi>w</mi><msub>1</msub>, ..., <mi>w</mi><msub>n</msub> <min> <mi>R</mi><msup>m</msup>, v = <msum from=0 to=n> <mi>a</mi><msub>k</msub><mi>w</mi><msub>k</msub></math>
<p>
Ici, toutes ces combinaisons donnent des résultats nets et précis.
Or, <span class="important">il existe des équations formulables avec des équations linéaires ne possédant aucune solution dans "R²"</span>.
</p>
<math>-2 * (1, 6) + -a * (1, 4) = 4 * (1, 2) <mequi> -2 * (1, 6) + -4 * (1, 2) = a * (1, 4) <mequi> (-2, -12) + (-4, -8) = a * (1, 4) <mequi> (-6, -20) = a * (1, 4) <mequi> a * 1 = -6 <mand> -4 * a = 20 <mequi> a = -6 <mand> a = -5 <mequi> <mequi> <mnotexists> a <min> <mi>R</mi><msup>2</msup></math></br>
<p>
Les combinaisons linéaires vont aussi nous permettre d'énumérer un fait très important.
En effet, <span class="important">pour un vecteur donné, il est possible d'obtenir tout un ensemble de nouveau vecteur avec une combinaison linéaire de ce vecteur</span>.
Ici, si l'on nomme ce vecteur "u", alors toutes les combinaisons linéaires possibles avec ce vecteur sont de la forme "a * u", où "a" est un nombre réel quelconque.
Par exemple, avec le vecteur "(1, 1)", vous pouvez formuler tous les vecteurs de la forme "a * (1, 1)", comme "(2, 2)", "(72, 72)", "(-0.1475, -0.1475)"...
</p>
<graphic><background_color white><base width=5 height=5><function expression="x" color=blue></function><line x_1=0 y_1=0 x_2=1 y_2=1 color=red></graphic>
<p>
Nous pouvons faire la même chose avec tous les vecteurs possibles.
Par exemple, prenons le vecteur "(1, 0)" : nous pouvons engendré "(3, 0)", "(pi, 0)" ou "(-0.000000000001, 0)".
Ici, <span class="important">nous pouvons obtenir une combinaison linéaire de nos deux vecteurs d'exemple : "(1, 1)" et "(1, 0)"</span>.
Nous obtenons donc une combinaison linéaire de la forme "a * (1, 1) + b * (1, 0)" avec "a" et "b" des nombres réels quelconques.
Ici, on peut se poser une question : <span class="important">quel est l'ensemble précis des vecteurs calculables via cette combinaison linéaire</span> ?
Quelques exemples peuvent s'obtenir en remplaçant "a" et "b" par des valeurs connues.
</p>
<math> 2 * (1, 1) + 1 * (1, 0) = (3, 2)</math></br>
<math> 0.5 * (1, 1) + 10 * (1, 0) = (10.5, 0.5)</math>
<p>
Prenons un exemple bien plus aléatoire : "(55, -44)".
Ici, nous pouvons poser une équation qui devrait nous permettre de trouver une solution, si elle existe.
</p>
<math>a * (1, 1) + b * (1, 0) = (55, -44) <mequi> (a * 1 + b * 1, a * 1 + b * 0) = (55, -44)</math></br>
<math><mequi> a + b = 55 <mand> a = -44 <mequi> b = 99 <mand> a = -44 <mequi> -44 * (1, 1) + 99 * (1, 0) = (55, -44)</math>
<p>
Vous pouvez prendre autant de valeurs que vous le voulez : vous trouverez toujours des valeurs fonctionnelles pour "a" et "b".
Et donc, une question se pose : <span class="important">est ce que tout "R²" est obtensible avec une simple combinaison linéaire de "(1, 1)" et "(1, 0)"</span> ?
Pour cela, considérons un vecteur "(x, y)" quelconque (faisant partie de "R²"), et voyons si, pour tout "x" et "y", nous pouvons trouver un "a" et un "b" fonctionnelle.
Posons cela sous la forme d'une équation.
</p>
<math>a * (1, 1) + b * (1, 0) = (x, y) <mequi> (a * 1 + b * 1, a * 1 + b * 0) = (x, y)</math></br>
<math><mequi> a + b = x <mand> a = y <mequi> b = x - y <mand> a = x <mequi> y * (1, 1) + (x - y) * (1, 0) = (x, y)</math>
<p>
Or, ici, "y" est bel et bien un réel, et "x - y" aussi : </span class="important">tout "(x, y)" possède des "a" et "b" fonctionnelles, tout "(x, y)" de "R²" peut s'écrire comme une combinaison linéaire de "(1, 1)" et "(1, 0)"</span>.
En procédant de manière similaire, <span class="important">on peut trouver plein de couples de vecteurs permettant aussi d'écrire tout "R²" : "(1, 1)" et "(1, 2)", "(1, 0)" et "(1, 1)", "(5, 100)" et "(5, 101)"...</span>.
À l'inverse, nous pouvons trouver des couples de vecteurs, comme "(1, 0)" et "(-3, 0)", pour lesquelles l'ensemble des vecteurs obtensible par combinaison linéaire n'est pas "R²".
Dans l'exemple pris ici, on démontre que l'ensemble des vecteurs obtenus ne représente pas "R²", car un élément de "R²" pris arbitrairement (bien qu'il y en ai une infinité possible) est absent de ces valeurs.
</p>
<math>a * (1, 0) + b * (-3, 0) = (x, y) <mequi> (a * 1 + b * -3, a * 0 + b * 0) = (x, y)</math></br>
<math><mequi> a + -3 * b = x <mand> y = 0 <mequi> (x, y) /= (1, 1)</math>
<p>
Dans tous les cas, nous formons un ensemble constitué d'une certaine quantité de vecteurs (ici 2), qui permettent d'obtenir des vecteurs possibles dans l'espace.
Généralement, pour parler d'ensemble de vecteurs, on utilie le terme de famille.
En mathématiques, <span class="important"><definition name="set_family"></span>.
Donc, nous définissons des ensembles de vecteurs, que nous nommons des familles de vecteurs, tel que leur combinaison linéaire puisse représenter des espaces quelconques.
</p>
<p>
Avec ces combinaisons linéaires, nous pouvons en déduire un théorème extrêmement important : <span class="important">l'ensemble des vecteurs calculables avec une combinaison linéaire d'un espace vectoriel quelconque avec une famille de vecteurs fixés et des coefficients "libres" dans le corps est un sous-espace vectoriel de cet espace vectoriel de départ</span>.
La démonstration de ce théorème est très simple, et est parfaitement algébrique.
Déjà, rappelons la définition d'une combinaison linéaire (et rappelons que, algébriquement parlant, tout élément d'une combinaison linéaire appartient à l'espace vectoriel, validant la première condition disant que l'ensemble du sous-espace vectoriel doit être inclut dans l'ensemble de l'espace).
</p>
<math><mi>w</mi><msub>0</msub>, <mi>w</mi><msub>1</msub>, ..., <mi>w</mi><msub>n</msub> <min> <mi>R</mi><msup>m</msup>, {<mi>a</mi><msub>0</msub>, <mi>a</mi><msub>1</msub>, ..., <mi>a</mi><msub>n</msub> <min> R, v = <msum from=0 to=n> <mi>a</mi><msub>k</msub><mi>w</mi><msub>k</msub>}</math>
<p>
Déjà, nous pouvons facilement prouver la stabilité par l'addition, en additionnant deux combinaisons de même famille de vecteurs entre elles.
</p>
<math><msum from=0 to=n> <mi>a</mi><msub>k</msub><mi>w</mi><msub>k</msub> + <msum from=0 to=n> <mi>b</mi><msub>k</msub><mi>w</mi><msub>k</msub> = <msum from=0 to=n> (<mi>a</mi><msub>k</msub>+<mi>b</mi><msub>k</msub>)<mi>w</mi><msub>k</msub> = <msum from=0 to=n> <mi>c</mi><msub>k</msub><mi>w</mi><msub>k</msub> <mand> <mi>c</mi><msub>0</msub>, <mi>c</mi><msub>1</msub>, ..., <mi>c</mi><msub>n</msub> <min> R</math>
<p>
Petit point important : ici, quand nous disons que les vecteurs sont fixés, nous voulons dire que, dans une opération de combinaisons linéaires, les vecteurs doivent rester les mêmes.
Cependant, cela n'empêche pas que cela fonctionne pour toute famille de vecteur possible (comme nous l'avons quantifier au début de la démonstration), choisi au préalable.
Finalement, on peut facilement prouver la stabilité par la multiplication par un scalaire.
</p>
<math>b * <msum from=0 to=n> <mi>a</mi><msub>k</msub><mi>w</mi><msub>k</msub>  = <msum from=0 to=n> (b * <mi>a</mi><msub>k</msub>)<mi>w</mi><msub>k</msub> = <msum from=0 to=n> <mi>d</mi><msub>k</msub><mi>w</mi><msub>k</msub> <mand> <mi>d</mi><msub>0</msub>, <mi>d</mi><msub>1</msub>, ..., <mi>d</mi><msub>n</msub> <min> R</math>
<p>
Les deux stabilités demandées sont bel et bien respectées : <span class="important">tout ensemble formé des vecteurs calculables avec une combinaison linéaire dans un espace vectoriel donne un sous-espace vectoriel de ce même ensemble</span>.
</p>
<p>
Avec ceci, nous pouvons introduire un concept primordial en algèbre linéaire : les sous-espaces vectoriels engendrés.
<span class="important"><definition name="sub_vector_space-span"></span>.
Une grande partie des théorèmes d'algèbre linéaire s'appuient entièrement sur ce concept.
Généralement, pour une famille de vecteurs "f", le sous-espace vectoriel engendré par "f" est nommé "Vect(f)".
</p>
<math>f = (<mi>u</mi><msub>1</msub>, <mi>u</mi><msub>2</msub>, ..., <mi>u</mi><msub>n</msub>) <min> <mi>E</mi><msup>n</msup></math></br>
<math><mforall> v <min> E, {v <min> Vect(f) <mequi> <mexists>(<mi>a</mi><msub>1</msub>, <mi>a</mi><msub>2</msub>, ..., <mi>a</mi><msub>n</msub>) <min> <mi>K</mi><msup>n</msup>, v = <mi>a</mi><msub>1</msub> * <mi>u</mi><msub>1</msub> + <mi>a</mi><msub>2</msub> * <mi>u</mi><msub>2</msub> + ... + <mi>a</mi><msub>n</msub> * <mi>u</mi><msub>n</msub>} </math>
<p>
Intéressons nous à ce concept de famille de vecteur.
Déjà, <span class="important>"il est possible qu'une famille de vecteur contienne des vecteurs qui s'engendre les uns par rapport aux autres, grâce à une combinaison linéaire</span>.
Par exemple, la famille de vecteur de "R²" "{(1, 1), (3, 3)}" contient un vecteur que l'on peut obtenir par combinaison linéaires d'autres (ici, "3 * (1, 1) = (3, 3)").
Ce genre de famille a un nom : les familles liées.
<span class="important"><definition name="vector_family-linked"></span>.
Par définition, si une famille est liée à cause d'un vecteur "u", alors "u" appartient à l'espace vectoriel engendré par la famille de départ à laquelle on enlève "u".
De même, tout multiple / combinaison linéaire de "u" appartient aussi à ce sous-espace vectoriel, et donc tout l'espace vectoriel engendré par "u" est aussi dans cet espace vectoriel.
Pour une famille liée "f", voici la formalisation mathématique du concept (où "g" est la famille où "u" est exclu).
</p>
<math>f = (<mi>u</mi><msub>1</msub>, <mi>u</mi><msub>2</msub>, ..., <mi>u</mi><msub>n</msub>) <min> <mi>E</mi><msup>n</msup></math></br>
<math><mexists> m <min> {1, 2, ..., n}, g = f \ {<mi>u</mi><msub>m</msub>}, g = {<mi>w</mi><msub>1</msub>, <mi>w</mi><msub>2</msub>, ..., <mi>w</mi><msub>n - 1</msub>} <min> <mi>E</mi><msup>n - 1</msup></math></br>
<math><mexists>(<mi>a</mi><msub>1</msub>, <mi>a</mi><msub>2</msub>, ..., <mi>a</mi><msub>n - 1</msub>) <min> <mi>K</mi><msup>n - 1</msup>, <mi>u</mi><msub>m</msub> = <mi>a</mi><msub>1</msub> * <mi>w</mi><msub>1</msub> + <mi>a</mi><msub>2</msub> * <mi>u</mi><msub>2</msub> + ... + <mi>a</mi><msub>n - 1</msub> * <mi>w</mi><msub>n - 1</msub></math>
<p>
À l'inverse, <span class="important"><definition name="vector_family-free"></span>.
Donc, on peut en déduire que <span class="important">le sous-espace vectoriel engendré par une famille de vecteur liée à cause d'un vecteur quelconque est égal à celui engendré par la famille de vecteur sans ce vecteur "u"</span>.
</p>
<math>Vect(f) = Vect(g)</math></br>
<graphic><base width=5 height=5><background_color white><line name=type_number() x_1=0 y_1=0 x_2=1 y_2=0 color=blue><line name=type_number() x_1=0 y_1=0 x_2=0 y_2=1 color=green><line name=type_number() x_1=0 y_1=0 x_2=1 y_2=1 color=red></graphic>
<p>
Le type de famille de vecteur le plus utile est probablement la famille génératrice.
<span class="important"><definition name="vector_family-generator"></span>.
Comme nous l'avons vue plus haut, c'est le cas de notre famille "{(1, 1), (1, 0)}" dans "R²".
Il est possible de compléter ce concept, pour en obtenir un nouveau très pratique : une base d'un espace vectoriel.
<span class="important"><definition name="vector_space_base"></span>.
Il s'agit de la donnée minimum nécessaire dans un espace vectoriel pour obtenir la valeur de chaque vecteur possible.
Il existe d'ailleurs autant de base qu'il existe de famille de vecteur libre et génératrice.
</p>
<math>f = (<mi>u</mi><msub>1</msub>, <mi>u</mi><msub>2</msub>, ..., <mi>u</mi><msub>n</msub>) <min> <mi>E</mi><msup>n</msup></math></br>
<math><mforall> v <min> E, <mexists>(<mi>a</mi><msub>1</msub>, <mi>a</mi><msub>2</msub>, ..., <mi>a</mi><msub>n</msub>) <min> <mi>K</mi><msup>n</msup>, v = <mi>a</mi><msub>1</msub> * <mi>u</mi><msub>1</msub> + <mi>a</mi><msub>2</msub> * <mi>u</mi><msub>2</msub> + ... + <mi>a</mi><msub>n</msub> * <mi>u</mi><msub>n</msub> </math>
<p>
Avec cela, nous allons "déterrer" un concept géométre que nous avons traité au chapitre3 : les coordonnées.
En effet, en algèbre linéaire, <span class="important">les coordonnées d'un vecteur quelconque dans une base quelconque représentent les valeurs des scalaires à multiplier à chaque élément de la base pour obtenir le vecteur recherché</span>.
En d'autres termes, il s'agit des scalaires dans la combinaisons linéaire permettant de forme le vecteur de départ via cette base.
Ici, les coordonnées dépendent évidemment du vecteur, mais aussi de la base choisie.
Généralement, <span class="important">les coordonnées d'un vecteur "v" dans une base "b" se note "v indice b"</span>.
Prenons un exemple simple dans "R2" : une base "B" représentant la base canonique, et la base "C" représentant une autre base.
</p>
<math>B = {(1, 0), (0, 1)}, C = {(1, 1), (0, 1)}</math></br>
<p>
Déjà, <span class="important">exprimer un vecteur avec des coordonnées donnée dans une base est extrêmement simple, puisque cela ne demande qu'une simple combinaison linéaire</span>.
Par exemple, trouvons les vecteurs de coordonnées "(4, 3)" dans la base "B" et "(5, 1)" dans la base "C".
</p>
<math><mi>v</mi><msub>1</msub> = 4 * (1, 0) + 3 * (0, 1) = (4, 3)</math></br>
<math><mi>v</mi><msub>2</msub> = 5 * (1, 1) + 1 * (0, 1) = (5, 6)</math></br>
<p>
Faisons l'opération inverse : <span class="important">trouver les coordonnées associées à un vecteur donné dans une certaine base</span>.
Il est très facile de trouver les expressions des vecteurs "(1, 0)" et "(1, 1)" dans la base canonique : il s'agit de "(1, 0)" et "(1, 1)".
</p>
<math><mi>(1, 0)</mi><msub>B</msub> = <mi>(1, 0)</mi>, <mi>(1, 1)</mi><msub>B</msub> = <mi>(1, 1)</mi></math></br>
<p>
Ici, le défi est de trouver les expressions des vecteurs "(1, 0)" et "(1, 1)" dans la base "C".
La meilleure façon est de résoudre un petit système, permettant de trouver les valeurs des scalaires.
C'est généralement de cette façon que nous fairons pour passer des coordonnées d'une base à une autre facilement.
</p>
<math>(1, 0) = <mi>a</mi><msub>1</msub> * (1, 1) + <mi>a</mi><msub>2</msub> * (0, 1) <mequi> 1 = <mi>a</mi><msub>1</msub> * 1 + <mi>a</mi><msub>2</msub> * 0 <mand> 0 = <mi>a</mi><msub>1</msub> * 1 + <mi>a</mi><msub>2</msub> * 1 <mequi> <mi>a</mi><msub>1</msub> = 1 <mand> <mi>a</mi><msub>2</msub> = -1</math></br>
<math>(1, 1) = <mi>a</mi><msub>1</msub> * (1, 1) + <mi>a</mi><msub>2</msub> * (1, 0) <mequi> <mi>a</mi><msub>1</msub> = 1 <mand> <mi>a</mi><msub>2</msub> = 0</math></br>
<p>
Donc, nous avons trouver les scalaires permettant de former ce vecteur dans la base "C" : il s'agit de ses coordonnées dans cette même base.
</p>
<math><mi>(1, 0)</mi><msub>C</msub> = <mi>(1, -1)</mi>, <mi>(1, 1)</mi><msub>C</msub> = <mi>(0, 1)</mi></math></br>
<p>
Si nous voulons formuler ces systèmes de manière plus précise, pour un vecteur "v" exprimé dans une base "b" :
</p>
<math>v = <msum from=1 to=n unknown=k><mi>a</mi><msub>k</msub><mi>b</mi><msub>k</msub></math>
<p>
Ou, avec chaque élément de "v" un à un :
</p>
<math><mi>v</mi><msub>1</msub> = <msum from=1 to=n unknown=k><mi>a</mi><msub>k</msub>(<mi>b</mi><msub>k</msub>)<msub>1</msub></math></br>
<math><mi>v</mi><msub>2</msub> = <msum from=1 to=n unknown=k><mi>a</mi><msub>k</msub>(<mi>b</mi><msub>k</msub>)<msub>2</msub></math></br>
<math><mi>v</mi><msub>i</msub> = <msum from=1 to=n unknown=k><mi>a</mi><msub>k</msub>(<mi>b</mi><msub>k</msub>)<msub>i</msub></math></br>
<p>
Un (autre) exemple dans "R2" est présent <a href="demo#4_1_0_2">juste ici</a>.
Ici, on sait que des coordonnées dans un espace de dimension fini "n" représentent donc "n" scalaires.
Donc, <span class="important">l'ensemble des coordonnées possibles dans une base d'un espace vectoriel sur un corps "K" de dimension "n" représentent le produit cartésien de "K", "n" fois</span>.
De plus, <span class="important">on peut en déduire que l'ensemble des coordonnées dans une base forme un espace vectoriel sur "K" lui aussi</span>.
La démonstration est immédiate : la somme des coordonnées de deux vecteurs donne les coordonnées du vecteur représentant la somme des vecteurs (stabilité de la somme), et multiplier les coordonnées par un scalaire donne les coordonnées du vecteur multiplié par ce même scalaire. 
</p>
<p>
Avec les bases, on peut définir un des concepts les plus importants des espaces vectoriels : la dimension.
<span class="important"><definition name="vector_space_dimension"></span>.
En effet, <span class="important">toutes les bases d'un espace vectoriel quelconque ont le même cardinal (la même taille)</span>.
En réalité, ce résultat n'est pas si facile à démontrer : il demande l'utilisation d'un lemme, nommé lemme de Steinitz (<a href="demo#4_1_0_0">démontré ici</a>), qui permet d'en déduire ce théorème (<a href="demo#4_1_0_1">démontré ici</a>).
</p>
<math>f = (<mi>u</mi><msub>1</msub>, <mi>u</mi><msub>2</msub>, ..., <mi>u</mi><msub>n</msub>) <min> <mi>E</mi><msup>n</msup></math></br>
<math>g = (<mi>v</mi><msub>1</msub>, <mi>v</mi><msub>2</msub>, ..., <mi>v</mi><msub>m</msub>) <min> <mi>E</mi><msup>n</msup></math></br>
<math>Vect(f) = Vect(g) = E <mequi> n = m</math>
<p>
Ici, bien que nous considérons "E" entier, <span class="important">une conséquence de ce théorème stipule que si "f" et "g" sont deux familles libres qui ne génèrent pas "E" entier, mais un sous-espace vectoriel "F" de "E" (et qu'elles génèrent le même, bien évidément), alors "m = n"</span>.
Pareil, ce théorème s'obtient en restreignant "E" au sous-espace vectoriel engendré : "f" et "g" en sont deux bases, et ont donc le même cardinal.
Cependant, dans ce cas, <span class="important">la réciproque est fausse, on peut trouver des familles "f" et "g" de même dimensions qui ne génère pas le même sous-espace vectoriel de "E"</span>.
De manière parfaitement logique, cela s'explique par le fait que passer à un sous-espace vectoriel requiert une implication dans le raisonnement ("F est un sous-espace vectoriel" implique que "toute base de "F ont le même cardinal", mais la réciproque de cette proposition ne veut rien dire), cassant l'équivalence (puisque l'on part d'un plus gros espace vectoriel, demandant de passer à un autre : le sous-espace vectoriel).
</p>
<math>Vect(f) = Vect(g) = F <mimp> n = m</math>

<h3>Les systèmes linéaires</h3>
<p>
Avant de traiter des systèmes linéaires, parlons des équations linéaires.
<span class="important"><definition name="linear_equation"></span>.
Par exemple, une équation linéaire dans un l'espace vectoriel "R²" sur "R" peut être :
</p>
<math>8 * (a, b) = (5, -6)</math>
<p>
Résoudre ce genre d'équation est très facile :
</p>
<math>k * (a, b) = (c, d) <mequi> (a, b) = (<mfrac><mi>c</mi><mi>k</mi></mfrac>, <mfrac><mi>d</mi><mi>k</mi></mfrac>)</math>
<p>
Avec ça, nous allons pouvoir parler des systèmes linéaires.
En fait, <span class="important"><definition name="linear_system"></span>.
Par exemple, ceci peut être un système linéaire, dans un espace vectoriel sur "R" :
</p>
<math>3 * u + 4 * v = 2</math></br>
<math>8 * u - 7 * v = 5</math>
<p>
La façon la plus simple de résoudre ce genre d'équation reste la substitution.
</p>
<math>3 * u + 4 * v = 2</math></br>
<math>8 * u - 7 * v = 5</math></br>
<math><mequi></math></br>
<math>8 * u = 7 * v + 5</math></br>
<math>3 * <mfrac><mi>7 * v + 5</mi><mi>8</mi></mfrac> + 4 * v = 2</math></br>
<math><mequi></math></br>
<math><mfrac><mi>53 * v + 15</mi><mi>8</mi></mfrac> = 2</math></br>
<math>8 * u = 7 * v + 5</math></br>
<math><mequi></math></br>
<math>v = <mfrac><mi>1</mi><mi>53</mi></mfrac></math></br>
<math>8 * u = 7 * <mfrac><mi>1</mi><mi>53</mi></mfrac> + 5</math></br>
<math><mequi></math></br>
<math>v = <mfrac><mi>1</mi><mi>53</mi></mfrac></math></br>
<math>u = <mfrac><mi>34</mi><mi>53</mi></mfrac></math></br>

<h3>Les applications linéaires</h3>
<p>
Les applications linéaires sont un objet élémentaire dans l'étude des espaces vectoriels.
De manière un peu complexe, <span class="important"><definition name="linear_application-morphism"></span>.
Il s'agit d'ici de la façon complexe de définir ce concept.
En effet, <span class="important"><definition name="linear_application"></span>.
Nous allons expliciter cette "similarité" dés maintenant.
Déjà, <span class="important">utiliser une application linéaire sur un vecteur "a + b" (avec "a" et "b" des vecteurs du premier espace vectoriel et "+" est la première loi du premier espace vectoriel) revient à appliquer la première loi du deuxième espace vectoriel entre l'image de "a" puis de "b" par cette même application linéaire</span>.
Donc, on peut "prédire" le résultat d'une opération dans le deuxième espace vectoriel grâce au premier espace vectoriel et une application linéaire.
Dans le cas d'une application linéaire "f" allant d'un espace vectoriel "E" vers un espace vectoriel "F" sur un corps "K", on note :
</p>
<math><mforall>a, b <min> E, f(a + b) = f(a) ~ f(b)</math>
<p>
Deuxièmement, <span class="important">utiliser une application linéaire sur un vecteur "c * a" (avec "a" un vecteur du premier espace vectoriel, "c" un scalaire provenant de "K" et "*" est la deuxième loi du premier espace vectoriel) revient à appliquer la deuxième loi du deuxième espace vectoriel entre "c" et l'image de "a" par cette même application linéaire</span>.
Encore une fois, on peut "prédire" le résultat d'une opération dans le deuxième espace vectoriel grâce au premier espace vectoriel et une application linéaire.
</p>
<math><mforall> c <min> K, <mforall>a <min> E, f(c * a) = c * f(a)</math>
<p>
Finalement, <span class="important">l'élément neutre du premier espace doit toujours avoir comme image l'élément neutre du deuxième espace par une application linéaire</span>.
Dire qu'une application entre deux espaces vectoriels respecte ces trois propriétés équivaut à dire que l'application est linéaire.
</p>
<p>
Prenons un exemple très simple : l'application allant de "R²" vers "R²", en transformant "(x, y)" en "(2 * x, 2 * y)".
Cette application est parfaitement linéaire, et on peut facilement le démontrer.
Déjà, la première propriété est respectée, et l'on peut le vérifier de manière algébrique.
</p>
<math><math><mforall>a, b <min> <mi>R</mi><msup>2</msup>, <mexists>x, x', y, y' <min> R, a = (x, y), b = (x', y'), f(a + b) = f(x + x', y + y') = 2 * (x + x', y + y') = 2 * (x, y) + 2 * (x', y') = f(a) + f(b)</math>
<p>
La deuxième propriété est aussi vraie, et ce n'est pas si difficile que ça à démontrer non plus.
</p>
<math><math><mforall>a, b <min> <mi>R</mi><msup>2</msup>, <mexists>x, y, c <min> R, a = (x, y), f(c * a) = f(c * x, c * y) = 2 * (c * x, c * y) = c * 2 * (x, y) = c * f(a)</math>
<p>
Finalement, "f(0, 0) = (0, 0)", <span class="important">donc cette application est belle est bien linéaire</span>.
On peut même généraliser ce résultat.
Effectivement, <span class="important">on peut facilement montrer que toutes applications transformant "(x, y)" en "(w * x, z * y)" avec "w" et "z" des scalaires quelconques est linéaire</span>.
Ce résultat <a href="demo#4_1_1_0">est démontré ici</a>, et est très similaire à la démonstration que nous venons juste de voir.
</p>
<p>
Un résultat intéressant avec les applications linéaires concerne les bases d'espaces vectoriels.
En effet, <span class="important">l'image d'une base de l'espace vectoriel de départ par une application linéaire représente une famille de vecteurs qui engendre l'image de l'application dans l'espace vectoriel d'arrivé</span>.
La démonstration de ce théorème (<a href="demo#4_1_1_1">présente ici</a>) utilise les résultats basiques de linéarité.
D'ailleurs, <span class="important">selon le théorème de la dimension, si l'image de l'application dans l'espace d'arrivée est de dimension inférieure ou égale à celle de l'espace de départ, alors l'image de la base génère tout l'espace d'arrivée, et si ils ont les mêmes dimensions, alors l'image de la base est une base de l'espace d'arrivée</span>.
</p>



<h2>Les matrices</h2>

<h3>Qu'est ce qu'est une matrice ?</h3>
<p>
Les matrices sont des objets très utilisées en algèbre linéaire.
En fait, <span class="important"><definition name="matrix"></span>.
Ici, il faut voir "tableau" comme un tableau à deux entrées, contenant donc des éléments de cet ensemble.
Par convention, ces objets se notent avec des lettres majuscules.
</p>
<math>A = <mrow><mo>[</mo><mtable><mtr><mtd><mi>1</mi></mtd><mtd><mi>2</mi></mtd></mtr><mtr><mtd><mi>3</mi></mtd><mtd><mi>4</mi></mtd></mtr></mtable><mo>]</mo></mrow></math>
<p>
Bien que très intuitive, cette définition manque de rigueur.
D'un point de vue rigoureux, <span class="important"><definition name="matrix-rigorous"></span>.
Généralement, on n'indique même pas les ensembles de taille "b" et "c", et l'on met directement la taille de "b" et "c" pour définir une matrice, où les ensembles utilisés pour indexer la matrice sont donc "{1, 2, ..., b}" et "{1, 2, ..., c}".
C'est dans ce genre de cas que la notation "tableau" devient extrêmement intuitive : le premier ensemble représente les lignes, et le deuxième représente les colonnes.
</p>
<p>
D'un point de vue "rigueur extrême", <span class="important">une matrice est donc une application allant du produit cartésien de ces deux ensembles vers des éléments de l'ensemble de départ</span>.
D'ailleurs, les images des valeurs des couples dans le produit cartésien des ensembles servant à l'indexage se note généralement avec la lettre utilisé pour noter la matrice en minuscule (pour notre matrice "A" il s'agirait donc de "a"), avec le couple du produit cartésien en indice.
C'est d'ailleurs comme ça que nous allons utiliser une notation pour accéder à un élément précis de la matrice.
En effet, <span class="important">un élément d'une matrice "A" à la ligne "y" et à la colonne "x" se note "A indice (x, y)"</span>.
L'ordre est important : ligne, puis colonne.
Reprenons notre matrice "A" :
</p>
<math>A = <mrow><mo>[</mo><mtable><mtr><mtd><mi>1</mi></mtd><mtd><mi>2</mi></mtd></mtr><mtr><mtd><mi>3</mi></mtd><mtd><mi>4</mi></mtd></mtr></mtable><mo>]</mo></mrow></math>
<p>
On peut facilement noter toutes les valeurs de "A".
</p>
<math><mi>a</mi><msub>1, 1</msub> = 1</math></br>
<math><mi>a</mi><msub>1, 2</msub> = 2</math></br>
<math><mi>a</mi><msub>2, 1</msub> = 3</math></br>
<math><mi>a</mi><msub>2, 2</msub> = 4</math>
<p>
Selon les tailles de "b" et "c", on parle de dimension de la matrice.
En effet, <span class="important"><definition name="matrix-dimension"></span>.
</p>
<p>
Comme pour beaucoup d'objets mathématiques, il est possible de faire des opérations avec ces matrices.
Si votre matrice représente des nombres, alors nous pouvons y définir une opération très utile : l'addition matricielle.
En effet, <span class="important">l'addition matricielle est une opération entre deux matrices d'exacte même dimension, où chaque élément d'indice "a, b" du résultat représente la somme des éléments d'indice "a, b" des deux matrices</span>.
Cette opération est bien définie car les ensembles de nombres ont généralement une addition de définie.
D'un point de vue définition, on peut noter la valeur de chaque élément de la matrice obtenue "C" pour l'addition de matrices "A" et "B" :
</p>
<math><mi>c</mi><msub>i, j</msub> = <mi>a</mi><msub>i, j</msub> + <mi>b</mi><msub>i, j</msub></math></br>
<p>
On peut même aller plus loin que ça.
Selon les propriétés algébriques de l'ensemble de départ des matrices, on peut leur en déduire des propriétés précises, et bien plus complexes qu'une simple addition.
Effectivement, <span class="important">si votre ensemble de départ peut former une structure algébrique précise possédant (au moins) une loi de composition interne, alors il est possible d'induire cette loi sur l'ensemble des matrices de même dimension, où chaque élément d'indice "a, b" du résultat représente la valeur après l'opération des éléments d'indice "a, b" des deux matrices</span>.
Cette propriété est immédiate de la façon même dont on écrit les éléments dans l'addition.
Grâce à ça, on peut rendre cette addition plus permissive.
</p>
<p>
Nous pouvons définir une autre opération assez utile : la multiplication par un scalaire.
Pour la définir, repassons dans un ensemble de matrices composés de nombres.
Ici, <span class="important">la multiplication par un scalaire est une opération entre un élément de l'ensemble de départ (nommé un scalaire) et une matrice, où chaque élément d'indice "a, b" du résultat représente le produit de l'élément d'indice "a, b" et du scalaire</span>.
Encore une fois, cette opération est bien définie car les ensembles de nombres ont généralement une addition de définie.
D'un point de vue définition, on peut noter la valeur de chaque élément de la matrice obtenue "C" pour la somme du scalaire "s" et de la matrice "A" :
</p>
<math><mi>c</mi><msub>i, j</msub> = s * <mi>a</mi><msub>i, j</msub></math></br>
<p>
Ici, il est difficile d'aller aussi loin que pour l'addition.
Généralement, <span class="important">pour que cela fonctionne, il est conseiller de définir vos matrices sur un ensemble représentant un anneau ou un corps, pour que les propriétés de la multiplication soient justement aussi permissive que dans des anneaux ou des corps</span>.
</p>
<p>
Vous l'aurez peut être deviné : <span class="important">avec ces deux lois, si l'ensemble de vos éléments de départ est un corps, nous pouvons définir l'ensemble des matrices comme un espace vectoriel de l'ensemble de départ</span>.
En effet, si l'ensemble de départ est un corps, l'addition matricielle est un groupe abélien, et la multiplication par un scalaire est associative, distributive et admet un neutre : il s'agit donc bien d'un espace vectoriel.
</p>
<p>
Parlons d'une autre opération, plus complexe mais très utile avec les matrices : le produit matriciel.
<span class="important"><definition name="matrix_product"></span>.
Soyons plus précis, avec une représentation graphique de deux matrices "A" et "B" de dimension "m, n" et "n, p", dont on va obtenir le produit matriciel.
</p>
<table>

<p>

En fait, <span class="important">pour un "c indice i, j", l'idée va être d'obtenir la somme du produit de chaque valeur de la "i-ième ligne" de la matrice à droite, et de la "j-ième" colonne de la matrice en haut</span>.
</p>
<math><mi>c</mi><msub>i, j</msub> = <msum from=1 to=n unknown=k><mi>a</mi><msub>k</msub><mi>b</mi><msub>k</msub></math>
<p>

</p>
<math>C = A * B</math>
<p>
Prenons un exemple avec deux matrices simples :
</p>

<p>
Ici, nous pouvons très facilement calculer chaque valeur de la matrice résultat, et donc obtenir la matrice résultat.
</p>
<math><mi>c</mi><msub>1, 1</msub> = <mi>a</mi><msub>1, 1</msub> * <mi>b</mi><msub>1, 1</msub> + <mi>a</mi><msub>1, 2</msub> * <mi>c</mi><msub>2, 1</msub> = 1 * 0 + 2 * 1 = 4</math></br>
<math><mi>c</mi><msub>1, 2</msub> = <mi>a</mi><msub>1, 1</msub> * <mi>b</mi><msub>1, 2</msub> + <mi>a</mi><msub>1, 2</msub> * <mi>c</mi><msub>2, 2</msub> = 1 * 2 + 2 * 0 = 2</math></br>
<math><mi>c</mi><msub>2, 1</msub> = <mi>a</mi><msub>2, 1</msub> * <mi>b</mi><msub>1, 1</msub> + <mi>a</mi><msub>2, 2</msub> * <mi>c</mi><msub>2, 1</msub> = 3 * 0 + 4 * 1 = 4</math></br>
<math><mi>c</mi><msub>2, 1</msub> = <mi>a</mi><msub>2, 1</msub> * <mi>b</mi><msub>1, 2</msub> + <mi>a</mi><msub>2, 2</msub> * <mi>c</mi><msub>2, 2</msub> = 3 * 2 + 4 * 0 = 6</math></br>
<math>C = <mrow><mo>[</mo><mtable><mtr><mtd><mi>4</mi></mtd><mtd><mi>2</mi></mtd></mtr><mtr><mtd><mi>4</mi></mtd><mtd><mi>6</mi></mtd></mtr></mtable><mo>]</mo></mrow></math></br>

<span class="important"><definition name="matrix-transpose"></span>.

<h3>Le lien applications linéaires - matrices</h3>
<p>
En algèbre linéaire, il existe un très gros lien entre les matrices et les applications linéaires.
En effet, <span class="important"><definition name="matrix-application"></span>.
Ici, une matrice prend les coordonnées d'un vecteur exprimé dans une base de départ, et permet d'obtenir les coordonnées de son image (via une application linéaire précise) dans une base d'arrivée.
Donc, <span class="important">la matrice représente un moyen de modifier correctement les coordonnées de départ en les coordonnées d'arrivée, en sachant la base de départ, d'arrivée et l'application linéaire auquelle elle correspond</span>.
Chose importante : <span class="important">une matrice peut donc aussi représente une application allant d'un espace de coordonnées vers un autre</span>.
Généralement, une matrice de ce type est notée avec un nom, sa base de départ et sa base d'arrivée (en indice) puis (entre parenthèse) l'application linéaire qu'elle représente.
</p>
<math><mi>M</mi><msub>b, c</msub>(f)</math>
<p>
Ici, nous allons essayer de faire une chose précise : construire intuitivement ce genre de matrice, pour comprendre comment elles fonctionnent.
Commençons par le cas le plus simple : passer de coordonnées d'un vecteur base quelconque aux coordonnées dans la base canonique.
Ici, <span class="important">ce passage représente l'application linéaire identité de l'espace vectoriel de départ, en changeant juste la base (base donnée vers base canonique, qui représente donc l'écriture directe du vecteur)</span>.
Imagineons que nous avons des coordonnées précises dans la base "b", que nous voudrions convertir dans la base canonique.
Nous pouvons donc facilement calculer son vecteur associé.
</p>
<math>v = <msum from=1 to=n unknown=k><mi>a</mi><msub>k</msub><mi>b</mi><msub>k</msub></math></br>
<math><mequi> <mi>v</mi><msub>i</msub> = <msum from=1 to=n unknown=k><mi>a</mi><msub>k</msub>(<mi>b</mi><msub>k</msub>)<msub>i</msub></math></br>
<p>
Or, nous savons que les coordonnées du vecteur dans la base canonique ont les mêmes valeurs que les éléments dans le vecteur.
Nous avons donc bel et bien dresser une façon de faire permettant de convertir facilement un vecteur.
Cependant, <span class="important">on peut aller plus loin, en essayant de rendre cette façon de faire graphique</span>.
Pour cela, <span class="important">nous allons utiliser des matrices</span>.
Présentons notre résultat comme ceci, en mettant les vecteurs de la base "b" en colonne :
</p>

Ici, nous avons dans nos cases en bas à droite les coordonnées du vecteur recherché.

En fait, <span class="important">il s'agit simplement du produit matriciel d'une matrice constituée des éléments de la base, et de la matrice représentant les coordonnées recherchées</span>.
C'est exactement le calcul que nous avons fait.
Chose intéressante : <span class="important">notre matrice ici à la même largeur que la taille de la base de départ "b", et la même hauteur que la taille de la base d'arrivée "c"</span>.
On peut donc lier la dimension de la matrice à la dimension de l'espace de départ et d'arrivée.
</p>
<p>
Tentons l'opération inverse : trouver les coordonnées dans une base donnée de vecteurs de coordonnées dans la base canonique.
Encore une fois, <span class="important">ce passage représente l'application linéaire identité de l'espace vectoriel de départ, en changeant juste la base (base canonique vers base donnée, qui représente donc l'écriture du vecteur dans la base donnée)</span>.
Pour rappel, si nous nommons notre base donnée "b", chaque valeur du vecteur "v" peut s'écrire :
</p>
<math><mi>v</mi><msub>i</msub> = <msum from=1 to=n unknown=k><mi>a</mi><msub>k</msub>(<mi>b</mi><msub>k</msub>)<msub>i</msub></math>
<p>
Donc, <span class="important">nous obtenons tout un système, contenant un nombre "n" correspondant à la taille de "b" d'équations, et autant d'inconnus</span>.
Nous pouvons donc en extraire les coordonnées nécessaires, représentant toutes les valeurs de "a" nécessaires.
<span class="important">Imagineons que nous faisons cela pour trouver les coordonnées des vecteurs de la base canonique dans la base "b"</span>.
Appelons ces vecteurs "c de i", où "i" représente le "i-ième" vecteur de la base.
</p>
<math><mi>c</mi><msub>i</msub> = <msum from=1 to=n unknown=k><mi>a</mi><msub>k</msub>(<mi>b</mi><msub>k</msub>)<msub>i</msub></math>
<p>
Ici, chaque coordonnées "c de i" représente les coordonnées du "i-èmes" vecteur de la base canonique dans la base "b".
Donc, <span class="important">exprimer des coordonnées de cette base dans la base canonique (avec les calculs vues plus haut) revient à exprimer des coordonnées de la base canonique en des coordonnées de la base "b"</span>.
En effet, ici, si vous cherchez la combinaison linéaire de vecteurs (ayant des coordonnées connues dans la base "b") pour obtenir leurs coordonnées dans la nouvelle base "c", vous obtenez en résultat les coordonnées du vecteur dans la base "b".
<span class="important">Il s'agit donc de la réciproque du calcul de coordonnées pour un vecteur donné dans une base</span>.
Nous pouvons donc écrire "v" comme une combinaison linéaire de vecteurs de "c" et de coordonnées, que nous nommerons "d".
</p>
<math><mi>v</mi><msub>i</msub> = <msum from=1 to=n unknown=k><mi>d</mi><msub>k</msub>(<mi>c</mi><msub>k</msub>)<msub>i</msub></math>
<p>
Avec ça, la conversion "base canonique" - "b" est assurée.
Comme pour plus haut, <span class="important">on peut écrire ce type de calcul avec une nouvelle matrice, non plus composer des éléments de "b", mais composer des éléments de "c"</span>.
</p>

<p>
Ici, nous avons construit un type très précis de matrice : des matrices de changement de base.
<span class="important"><definition name="matrix-basis"></span>.
Ces matrices représentent donc l'application identité, dans des bases différentes d'un même espace vectoriel.
De plus, <span class="important">comme deux bases d'un même espace vectoriel génère le même espace vectoriel, ont a donc une application allant de l'espace de départ vers l'espace de départ : il s'agit donc d'une bijection, et donc d'un automorphisme linéaire</span>.
Par exemple, dans "R²", la matrice passant de la base "(1, 0), (1, 1)" à la base canonique serait :
</p>
<math>A = <mrow><mo>[</mo><mtable><mtr><mtd><mi>1</mi></mtd><mtd><mi>1</mi></mtd></mtr><mtr><mtd><mi>0</mi></mtd><mtd><mi>1</mi></mtd></mtr></mtable><mo>]</mo></mrow></math>
<p>
Finalisons cette construction intuitive, avec un exemple plus global.
Considérons une application linéaire "f", allant d'un espace vectoriel "E" vers un espace vectoriel "F" (sur un corps "K").
Ici, nous allons considérer les bases comme quelconques.
Considérons "b" une base de "E".
</p>
<math>b = (<mi>b</mi><msub>1</msub>, <mi>b</mi><msub>2</msub>, ..., <mi>b</mi><msub>n</msub>) <min> <mi>E</mi><msup>n</msup></math></br>
<math><mforall> v <min> E, <mexists>(<mi>a</mi><msub>1</msub>, <mi>a</mi><msub>2</msub>, ..., <mi>a</mi><msub>n</msub>) <min> <mi>K</mi><msup>n</msup>, v = <mi>a</mi><msub>1</msub> * <mi>b</mi><msub>1</msub> + <mi>a</mi><msub>2</msub> * <mi>b</mi><msub>2</msub> + ... + <mi>a</mi><msub>n</msub> * <mi>b</mi><msub>n</msub> </math>
<p>
Selon un théorème vue plus haut, <span class="important">l'image des vecteurs de "b" forme une famille génératrice de l'image de "f"</span>.
Appelons cette image "c".
</p>
<math>c = (<mi>c</mi><msub>1</msub>, <mi>c</mi><msub>2</msub>, ..., <mi>c</mi><msub>n</msub>) <min> <mi>F</mi><msup>m</msup></math></br>
<math><mforall> v <min> F, {v <min> Im(f) <mequi> <mexists>(<mi>a</mi><msub>1</msub>, <mi>a</mi><msub>2</msub>, ..., <mi>a</mi><msub>n</msub>) <min> <mi>K</mi><msup>n</msup>, v = <mi>a</mi><msub>1</msub> * <mi>c</mi><msub>1</msub> + <mi>a</mi><msub>2</msub> * <mi>c</mi><msub>2</msub> + ... + <mi>a</mi><msub>n</msub> * <mi>c</mi><msub>n</msub>}</math>
<p>
Donc, pour tout vecteur "v" de "E", "f(v)" peut s'écrire comme une combinaison linéaire de vecteurs de "c".
Par linéarité de "f", les coordonnées de "v" dans "b" sont les mêmes que les "coordonnées" de "f(v)" dans "c".
</p>
<math>f(v) = f(<msum from=1 to=n unknown=k><mi>a</mi><msub>k</msub> * <mi>b</mi><msub>k</msub>) = <msum from=1 to=n unknown=k>f(<mi>a</mi><msub>k</msub> * <mi>b</mi><msub>k</msub>) = <msum from=1 to=n unknown=k><mi>a</mi><msub>k</msub> * f(<mi>b</mi><msub>k</msub>) = <msum from=1 to=n unknown=k><mi>a</mi><msub>k</msub> * <mi>c</mi><msub>k</msub></math>
<p>
Nous obtenons donc une expression de "f(v)" dans "c".
Cependant, nous avons un problème : <span class="important">nous ne savons pas si "c" est une base de "Im(f)" ou pas, car rien ne nous dit qu'elle est libre</span>.
Or, rappelons nous que tout vecteur d'un espace vectoriel peut s'exprimer avec une base de cet espace vectoriel, et que tout espace vectoriel possède au moins une base.
Donc, chaque "c" peut s'exprimer comme coordonnées dans une base de l'espace vectoriel d'arrivée, que nous nommerons "d".
Nous pouvons définir cette base comme bon nous semble, mais utilisons ici la façon la plus "globale".
</p>
<math><mexists>(<mi>e</mi><msub>1</msub>, <mi>e</mi><msub>2</msub>, ..., <mi>e</mi><msub>m</msub>) <min> <mi>K</mi><msup>n</msup>, <mi></mi>c<msub>i</msub> = <msum from=1 to=m unknown=k><mi>e</mi><msub>k</msub> * <mi>d</mi><msub>k</msub></math>
<p>

</p>

Donc, <span class="important">donc, il s'agit de la conversion des "coordonnées" dans la famille "c" vers les coordonnées dans la base "b"</span>.

<h3>Le lien matrice - application linéaire</h3>
<p>
Nous avons donc vue que toute application linéaire possède une matrice associée.
Cependant, une question peut se poser : est-ce que toutes les matrices peuvent être associées à des application linéaires ou pas.
La réponse est, en réalité, très simple à obtenir.
En effet, le produit d'une matrice quelconque et de coordonnées donne une combinaison linéaire des colonnes de la matrice, qui peut donc représenter une combinaison linéaire des coordonnées présentes dans la matrice.
Or, <span class="important">la transformation de coordonnées en une combinaison linéaire d'autres coordonnées forme une application linéaire (comme vu plus haut) : il s'agit donc bel et bien d'une application linéaire</span>.
Donc, toute matrice représente une application linéaire.
D'ailleurs, <span class="important">on peut très facilement trouver une famille génératrice de l'image d'arrivée : les colonnes de la matrice</span>.
</p>
<math>C = ((1, 0), (0, 1))</math></br>
<math>M = <mrow><mo>[</mo><mtable><mtr><mtd><mi>1</mi></mtd><mtd><mi>2</mi></mtd></mtr><mtr><mtd><mi>3</mi></mtd><mtd><mi>4</mi></mtd></mtr></mtable><mo>]</mo></mrow></math></br>
<math>f : R² <mto> R², f((x, y)) = x * (1, 3) + y * (2, 4) <mimp> <mi>Mat</mi><msub>C, C</msub>(f) = M</math>
<p>
Avec ça, on peut en déduire un théorème assez intéressant : <span class="important">toute matrice inversible représente un automorphisme linéaire sur les coordonnées</span>.
En effet, si une matrice est inversible, il existe une autre matrice dont le produit matriciel de ces deux matrices donne la matrice identité.
Or, ces deux matrices représentent donc deux applications linéaires, tel que la deuxième est la réciproque de la première.
Comme vue au chapitre 1, <span class="important">si une application admet une application réciproque, elle est donc bijective : l'application linéaire l'est donc aussi, ce qui en fait un isomorphisme linéaire</span>.
La réciproque est donc immédiate : <span class="important">si une application linéaire est un isomorphisme linéaire, elle admet donc une application réciproque, donc la matrice associée est la matrice inverse de la matrice associé à l'application de départ</span>.
Il s'agit donc d'une équivalence.
Bien évidemment, dans ces deux cas, il faut que les bases de départ et d'arrivée restent invariantes.
Finalement, <span class="important">comme la matrice est inversible, elle est carrée, et représente donc un endomorphisme de coordonnées : elle est donc un endomorphisme de coordonnées et un isomorphisme (qui représente aussi un isomorphisme pour les coordonnées), et donc un automorphisme de coordonnées</span>.
</p>
<math><mforall> A <min> M(n), {A <min> GL(n)} <mequi> {<mi>f</mi><msub>A</msub> est un automorphisme linéaire}</math>
<p>
Cependant, il existe d'autres opérations sur les matrices, qui permettent de trouver d'autres propriétés utiles sur ces dernières.
Ici, <span class="important">une des opérations les plus importantes de ce style revient à trouver le rang d'une matrice</span>.
Pour cela, nous allons procéder par des opérations qui ne modifient pas le rang de la matrice.
</p>
<p>
Déjà, il existe un type précis de produit matriciel qui ne modifie pas le rang de la matrice.
En effet, <span class="important">faire le produit matriciel à droite ou à gauche d'une matrice avec une matrice inversible ne change pas le rang de la matrice de départ</span>.
L'explication est toute simple : faire ce produit matriciel revient à obtenir la matrice associé à l'application composé de notre matrice de départ et de notre matrice inversible.
Or, l'application de la matrice inversible est un isomorphisme linéaire.
En fait, <span class="important">l'espace des coordonnées de départ et d'arrivée d'un isomorphisme linéaire sont toujours les mêmes</span>.
Donc, un isomorphisme linéaire ne modifie pas l'espace des coordonnées d'arrivée / espace des coordonnées de départ (et donc, pas sa dimension non-plus), et sa matrice associée non-plus.
Donc, faire le produit matriciel à droite ou à gauche d'une matrice avec une matrice inversible ne change pas le rang de la matrice de départ.
</p>
<p>
Une autre de ces opérations est la permutation de colonnes.
En effet, <span class="important">permuter les colonnes d'une matrice revient à permuter deux vecteurs dans la famille génératrice de l'image de l'application, ce qui peut demander de changer les bases de départ / arrivé, mais ne change pas les indépendances linéaires possibles dans cette famille génératrice, et donc ce qui ne change pas le rang de la matrice (et ce qui ne change pas l'espace engendré non-plus)</span>.
</p>
<math>M = <mrow><mo>[</mo><mtable><mtr><mtd><mi>1</mi></mtd><mtd><mi>2</mi></mtd></mtr><mtr><mtd><mi>3</mi></mtd><mtd><mi>4</mi></mtd></mtr></mtable><mo>]</mo></mrow></math></br>
<math>N = <mrow><mo>[</mo><mtable><mtr><mtd><mi>2</mi></mtd><mtd><mi>1</mi></mtd></mtr><mtr><mtd><mi>4</mi></mtd><mtd><mi>3</mi></mtd></mtr></mtable><mo>]</mo></mrow></math></br>
<math>Rg(M) = Rg(n), M . (1, 0) = N . (0, 1)</math>
<p>
Une autre opération de ce genre est la multiplication de tous les éléments d'une colonne par un scalaire non-nul précis.
En effet, <span class="important">multiplier tous les éléments d'une colonne par un scalaire nol-nul revient intuitivement à agrandir / rétricir cette colonne, ce qui ne change pas la façon dont elle est linéairement indépendante (ou pas) des autres colonnes (et ce qui ne change ni la dimension, ni l'espace engendré par la matrice)</span>.
</p>
<math>M = <mrow><mo>[</mo><mtable><mtr><mtd><mi>1</mi></mtd><mtd><mi>1</mi></mtd></mtr><mtr><mtd><mi>1</mi></mtd><mtd><mi>0</mi></mtd></mtr></mtable><mo>]</mo></mrow></math></br>
<math>N = <mrow><mo>[</mo><mtable><mtr><mtd><mi>2</mi></mtd><mtd><mi>1</mi></mtd></mtr><mtr><mtd><mi>2</mi></mtd><mtd><mi>0</mi></mtd></mtr></mtable><mo>]</mo></mrow></math></br>
<math>Rg(M) = Rg(n), M . (2, 0) = N . (1, 0)</math>
<p>
Finalement, nous pouvons trouver une dernière opération utile de ce genre : additionner / soustraire des colonnes de la matrice par d'autres colonnes de la matrice.
En effet, <span class="important">soustraire / additionner des colonnes à une autre colonne ne change pas la façon dont elle est linéairement indépendante (ou pas) des autres colonnes (si elle l'est au départ, alors elle le restera même après cette opération), ce qui ne change ni la dimension, ni l'espace engendré par la matrice</span>.
Avec ces 3 opérations, on peut déjà commencer à entrevoir la méthode générale pour travailler avec des matrices.
</p>
<math>M = <mrow><mo>[</mo><mtable><mtr><mtd><mi>1</mi></mtd><mtd><mi>5</mi></mtd></mtr><mtr><mtd><mi>2</mi></mtd><mtd><mi>10</mi></mtd></mtr></mtable><mo>]</mo></mrow></math></br>
<math>N = <mrow><mo>[</mo><mtable><mtr><mtd><mi>5</mi></mtd><mtd><mi>5</mi></mtd></mtr><mtr><mtd><mi>10</mi></mtd><mtd><mi>10</mi></mtd></mtr></mtable><mo>]</mo></mrow></math></br>
<math>P = <mrow><mo>[</mo><mtable><mtr><mtd><mi>5</mi></mtd><mtd><mi>0</mi></mtd></mtr><mtr><mtd><mi>10</mi></mtd><mtd><mi>0</mi></mtd></mtr></mtable><mo>]</mo></mrow></math></br>
<math>Rg(M) = Rg(N) = Rg(P)</math>
<p>
En effet, ici, <span class="important">on peut voir que le vecteur (5, 10) seul génère un espace vectoriel de même dimension que l'espace vectoriel engendré par notre matrice de départ (comme l'autre vecteur est nul, on peut le passer) : elle est de rang 1</span>.
Avec ça, nous pouvons introduire un algorithme très utile en algèbre linéaire : l'échelonnement vertical.
<span class="important"><definition name="echelon_vertical"></span>.
Cette méthode s'inspire d'une autre méthode plus connue (la méthode du pivot de Gauss), que nous verrons plus tard.
En tout cas, l'idée de l'échelonnement vertical est assez simple.
Premièrement, nous allons repérer la colonne qui contient comme coefficient le plus haut le premier coefficient (en partant de la gauche) non nul, et nous allons permuter cette colonne avec la première colonne.
Si la première colonne admet un coefficient le plus haut non-nul, alors pas besoin de faire cette étape.
Deuxièmement, <span class="important">nous allons effectuer les opérations nécessaire (multiplication, puis addiction / soustraction) sur les colonnes à droite de notre colonne actuelle pour que tous les coefficients à droite du coefficient le plus haut valent 0</span>.
Attention : il ne faut pas modifier que ce coefficient, mais toute la colonne en même temps.
Finalement, <span class="important">on ré-itère l'opération, en ne choisissant plus le coefficient le plus haut, mais le deuxième plus haut, et en ne commençant pas par la première colonne, mais par la deuxième</span>.
En suite, <span class="important">faites cela jusqu'à que vous arriviez en bas de votre matrice, où tout à droite de votre matrice</span>.
</p>
<p>
Ici, <span class="important">toutes ces opérations ne modifient pas le rang de la matrice, et en plus ne modifient pas l'espace engendré par la matrice</span>.
C'est cela qui rend cet algorithme si efficace : vous pouvez déduire ces deux propriétés dans une matrice de départ grâce à lui.
Prenons un petit exemple avec une matrice de rang 2, mais où il est difficile de le voir directement :
</p>
<math><mrow><mo>[</mo><mtable><mtr><mtd><mi>1</mi></mtd><mtd><mi>-2</mi></mtd><mtd><mi>0</mi></mtd></mtr><mtr><mtd><mi>3</mi></mtd><mtd><mi>-10</mi></mtd><mtd><mi>4</mi></mtd></mtr><mtr><mtd><mi>2</mi></mtd><mtd><mi>-9</mi></mtd><mtd><mi>5</mi></mtd></mtr></mtable><mo>]</mo></mrow></math></br>
<p>
Ici, la première étape ne sert à rien : le coefficient le plus haut de la première colonne est non-nul.
Passons à la deuxième : nous devons soustraire à la deuxième colonne la première colonne multipliée par "-2".
En suite, nous devons soustraire la troisième colonne par la première colonne multipliée par "0". 
</p>
<math><mrow><mo>[</mo><mtable><mtr><mtd><mi>1</mi></mtd><mtd><mi>0</mi></mtd><mtd><mi>0</mi></mtd></mtr><mtr><mtd><mi>3</mi></mtd><mtd><mi>-4</mi></mtd><mtd><mi>4</mi></mtd></mtr><mtr><mtd><mi>2</mi></mtd><mtd><mi>-5</mi></mtd><mtd><mi>5</mi></mtd></mtr></mtable><mo>]</mo></mrow></math></br>
<p>
maintenant, passons à la partie 2.
Encore une fois, la première étape ne sert à rien : le coefficient le "deuxième" plus haut de la deuxième colonne est non-nul.
Passons à la deuxième : nous devons soustraire à la troisième colonne la deuxième colonne multipliée par "-1".
</p>
<math><mrow><mo>[</mo><mtable><mtr><mtd><mi>1</mi></mtd><mtd><mi>0</mi></mtd><mtd><mi>0</mi></mtd></mtr><mtr><mtd><mi>3</mi></mtd><mtd><mi>-4</mi></mtd><mtd><mi>0</mi></mtd></mtr><mtr><mtd><mi>2</mi></mtd><mtd><mi>-5</mi></mtd><mtd><mi>0</mi></mtd></mtr></mtable><mo>]</mo></mrow></math></br>
<p>
Or, cette matrice génère donc un espace vectoriel généré par deux vecteurs de coordonnées, et a le même rang et génère le même espace vectoriel que notre matrice de départ : notre matrice de départ est donc de rang 2, et son image est engendrée par les deux vecteurs de coordonnée "(1, 3, 2)" et "(0, -4, -5)" dans la base d'arrivée.
</p>
<p>
Bon, ici, nous travaillons sur les colonnes.
Or, <span class="important">il est généralement conseillé de plutôt travailler sur des lignes</span>.
Pour comprendre comment cela est possible, il existe une visualisation mentale très simple, permettant de le comprendre et de le démontrer très facile.
Consédirons une matrice quelconque, et sa matrice transposée.
Par définition, dans une matrice transposée, les lignes de notre matrice de départ deviennent les colonnes de la transposée.
Donc, les lignes de la matrice de départ génèrent l'exact même espace que les colonnes de la matrice transposée.
Or, comme on l'a vue, les opérations sur les colonnes ne modifient ni le rang, ni l'espace engendré de la matrice.
La matrice transposée obéit aussi à cette règle : les opérations sur ces colonnes ne modifie pas son espace engendré.
Or, les colonnes de la matrice transposée représentent les lignes de la matrice de départ : les opérations sur les colonnes de la matrice transposée reviennent à faire des opérations sur les lignes de la matrice de départ, qui laissent donc l'espace qu'elles engendrent invariant.
Donc, <span class="important">l'espace engendré par les lignes de la matrice de départ ne change pas quand on y applique les mêmes opérations que celles sur les lignes</span>.
</p>
<p>
Nous pouvons aussi déduire un autre résultat très intéressant en algèbre linéaire : <span class="important">l'espace engendré par les lignes a l'exacte même dimension que l'espace engendré par les colonnes</span>.
En fait, <span class="important">ce résultat est parfaitement équivalent à dire qu'une matrice a le même rang que sa matrice transposée</span>.
La démonstration est particulièrement ennuyante à écrire, mais sa compréhension brute est assez simple.
Commençons par le cas des matrices carrées.
Pour une matrice carrée, <span class="important">si cette matrice ne peut pas être raccourcie par échelonnement vertical et sa transposée non plus, elle a donc le même rang que sa matrice de départ, et donc les lignes et les colonnes ont elles aussi le même rang</span>.
Cependant, si par échelonnement vertical, on obtient (de notre matrice carrée) une matrice moins large, la situation se complique.
Or, <span class="important">nous pouvons transposer cette matrice (pour en faire une plus large que haute), l'échelonner verticalement, pour en obtenir une nouvelle matrice carrée, et y appliquer le même constat</span>.
Dans tous les cas, si on trouve la matrice carrée finale, on sait qu'elle a la même dimension que sa transposée.
Cependant, la matrice non échelonnée a elle aussi le même rang que la matrice carrée, et sa transposée.
Dans ce cas, <span class="important">sa transposée représente une modification par des opérations sur la matrice carrée transposée qui ne changent pas le rang : elle est donc de même rang que sa matrice de départ</span>.
</p>











<h2>Les espaces de Hilbert et la géométrie euclidienne</h2>

<h3>Les espaces vectoriels normés</h3>
<p>
Grâce aux concepts que nous allons introduire maintenant, nous allons pouvoir nous rapprocher vers une formalisation précise de la géométrie.
Déjà, commençons à nous intéresser au concept de norme.
En fait, <span class="important"><definition name="vectoriel_space-norm"></span>.
Il faut faire attention : cette application n'est pas un opérateur entre deux vecteurs quelconques : elle n'est donc pas une distance.
Pour qu'une application allant de l'espace vectoriel à l'ensemble des réels positifs soit une norme, elle doit respecter quelques propriétés, similaire aux propriétés que doivent respecter une distance.
Pour les expliciter, nous allons appeler cette norme "N", et l'espace vectoriel "E".
Déjà, <span class="important">cette application est séparée : si un vecteur a pour image "0", il est obligatoirement le vecteur nul de l'espace</span>.
</p>
<math>N(x) = 0 <mimp> <mi>0</mi><msub>E</msub></math>
<p>
En suite, <span class="important">cette application est homogène : si un vecteur a pour image "a", la multiplication de ce vecteur par un scalaire "b" a pour image "|b| * a"</span>.
Ici, il est important de prendre la valeur absolue de "b", pour ne pas qu'un signe "-" vienne rendre le résultat négatif, ce qui est impossible par définition de cette application.
</p>
<math>N(x) = a <mimp> N(b * x) = |b| * a</math>
<p>
Finalement, <span class="important">cette application respecte une inégalité triangulaire : la norme de la somme de deux vecteurs "x" et "y" est inférieur ou égale à la somme de la norme du vecteur "x" et de la norme du vecteur "y"</span>.
Comme pour les espaces métriques, cette inégalité triangulaire traduit que le chemin entre "0" et "x + y" est plus court que celui entre "x", 0 puis "y" : le chemin direct est plus court que de passer par un autre point.
</p>
<math><mforall>x, y <min> E, N(x + y) <mltequal> N(x) + N(y)</math>
<p>
Donc, <span class="important"><definition name="vector_space-normed"></span>.
Ici, un exemple s'impose pour mieux illustrer tout ça : prenons l'espace vectoriel "R²" du corps formé par "R".
La question ici : comment définir une "norme" d'un élément de "R²" ?
En fait, nous allons essayer de représenter cette norme comme la distance entre le vecteur que nous cherchons et le vecteur nul (que nous pourrions théoriquement obtenir avec une mesure à la règle en géométrie pure), de manière algébrique.
La façon généralement utilisée est la méthode utilisant le théorème de Pythagore : nous n'allons pas encore l'utliser pour l'instant.
</p>
<p>
L'espace vectoriel normé de dimension fini le plus simple à définir est probablement celui muni d'une norme précise : la norme 1.
<span class="important"><definition name="vector_space-normed_1"></span>.
Pour que cela marche, il faut que le corps de départ soit muni d'une "valeur absolue", comme dans "Q" ou dans "R".
On peut asssez facilement démontre que l'application associant à chaque vecteur de l'espace la somme des valeurs absolues de ses coefficients est une norme.
</p>
<math>f : <mi>K</mi><msup>n</msup> <mto> R, (a, b, ..., c) <min> <mi>K</mi><msup>n</msup> <mto> |a| + |b| + ... + |c| </math>
<p>
Déjà, la séparation est évidente : si une somme de valeurs positives est nulle, alors tous les éléments de la somme le sont aussi.
</p>
<math>|a| + |b| + ... + |c| = 0 <mimp> a = 0, b = 0... c = 0 <mequi> (a, b, ..., c) = (0, 0, ..., 0)</math>
<p>
L'homogénéité est, elle aussi, assez évidente, et se déduit par une simple factorisation.
</p>
<math>f((a, b, ..., c)) = d <mand> f(q * (a, b, ..., c)) = e <mimp> |a| + |b| + ... + |c| = d <mand> |q * a| + |q * b| + ... + |q * c| = e <mimp> |q| * |a| + |q| * |b| + |q| * |c| = |q| * (|a| + |b| + ... + |c|) = |q| * d = e <mimp> f(q * (a, b, ..., c)) = |q| * f((a, b, ..., c))</math>
<p>
L'inégalité triangulaire n'est pas bien plus compliqué à démontrer, mais demande un peu plus d'éléments.
</p>
<math>f((a, b, ..., c)) = d <mand> f((e, g, ..., h)) = i <mimp> |a| + |b| + ... + |c| = d <mand> |e| + |g| + ... |h| = i <mimp> f((a, b, ..., c) + (e, g, ..., h)) = |a + e| + |b + g| + ... + |c + h|</math></br>
<math>|a + e| <mltequal> |a| + |e| <mand> |b + g| <mltequal> |b| + |g| <mand> ... <mand> |c + h| <mltequal> |c| + |h| <mimp> |a + e| + |b + g| + ... + |c + h| <mltequal> |a| + |e| + |b| + |g| + .. + |c| + |h| <mimp> f((a, b, ..., c) + (e, g, ..., h)) <mltequal> f((a, b, ..., c)) + f((e, g, ..., h))</math>
