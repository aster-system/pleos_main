<h1>Chapitre 3 - démonstrations</h1>



<h2>Les nombres réels</h2>

<h3 id="real_addition">L'opération d'addition</h3>
<h4>3.0.0.0</h4>
<p>
Démontrons que l'addition de deux nombres réel "a" et "b" donne un nombre réel (et donc, que l'addition est stable dans "R").
Soit deux nombres réel "a" et "b" :
</p>
<math><mforall>a, b <min> R, <mexists>A, B, C, D <min> P(Q), a = (A, B), b = (C, D)</math>
<p>
Admettons un objet "g", représentant "a + b".
Nous pouvons l'écrire avec la formule de l'addition de nombres réels.
</p>
<math>g = a + b <mequi> <mexists> E, F <min> P(Q), g = (E, F), <mforall> f <min> F, <mforall>c <min> A, <mforall>d <min> C, f <mgt> c + d <mand> <mforall> e <min> E, <mforall> f <min> F, e <mlt> f</math>
<p>
Par définition, "E" et "F" sont des sous-parties de "Q", et la relation d'ordre entre leurs éléments prouve que "E" et "F" sont complémentaires.
Donc, "g" est un nombre réel.
</p>

<h4>3.0.0.1</h4>
<p>
Démontrons que l'ordre d'addition de deux trois nombres réels "a", "b" et "c" n'importe pas (et donc, que l'addition est associative dans "R").
Soit trois nombres réels "a", "b" et "c" :
</p>
<math><mforall>a, b, c <min> R, <mexists>A, B, C, D, E, F <min> P(Q), a = (A, B), b = (C, D), c = (E, F)</math>
<p>
Admettons un objet "j", représentant "(a + b) + c".
Nous pouvons l'écrire avec la formule de l'addition de nombres réels.
</p>
<math>j = (a + b) + c <mequi> <mexists>G, H <min> P(Q), j = (G, H) <mequi> <mforall> h <min> H, <mforall>c <min> A, <mforall>d <min> C, <mforall> e <min> E, h <mgt> (c + d) + e <mand> <mforall> g <min> G, <mforall> h <min> H, g <mlt> h</math>
<p>
Comme l'addition est associative dans "Q", on peut modifier l'ordre des additions pour définir "j".
</p>
<math>j = (a + b) + c <mequi> <mexists>G, H <min> P(Q), j = (G, H) <mequi> <mforall> h <min> H, <mforall>c <min> A, <mforall>d <min> C, <mforall> e <min> E, h <mgt> (c + d) + e = c + (d + e) <mand> <mforall> g <min> G, <mforall> h <min> H, g <mlt> h <mequi> j = a + (b + c)</math>
<p>
Or, cela consiste à dire que l'on peut aussi modifier l'ordre des additions dans "R".
Donc, "(a + b) + c" est égal à "a + (b + c)".
L'opération d'addition est associative dans "R".
</p>
<math><mforall>a, b, c <min> R, (a + b) + c = a + (b + c)</math>

<h4>3.0.0.2</h4>
<p>
Démontrons que l'ordre de deux opérandes "a" et "b" autour d'une addition de nombres réels n'importe pas.
Soit deux nombres réels "a" et "b" :
</p>
<math><mforall>a, b <min> R, <mexists>A, B <min> P(Q), a = (A, B), b = (C, D)</math>
<p>
Admettons un objet "j", représentant "a + b".
Nous pouvons l'écrire avec la formule de l'addition de nombres réels.
</p>
<math>j = a + b <mequi> <mexists>G, H <min> P(Q), j = (G, H) <mequi> <mforall> h <min> H, <mforall>c <min> A, <mforall>d <min> C, <mforall> e <min> E, h <mgt> a + b <mand> <mforall> g <min> G, <mforall> h <min> H, g <mlt> h</math>
<p>
Comme l'addition est commutative dans "Q", on peut modifier l'ordre des opérandes pour définir "j".
</p>
<math>j = a + b <mequi> <mexists>G, H <min> P(Q), j = (G, H) <mequi> <mforall> h <min> H, <mforall>c <min> A, <mforall>d <min> C, <mforall> e <min> E, h <mgt> a + b = b + a <mand> <mforall> g <min> G, <mforall> h <min> H, g <mlt> h <mequi> j = b + a</math>
<p>
Or, cela consiste à dire que l'on peut aussi modifier l'ordre des opérandes dans une addition dans "R".
Donc, "a + b" est égal à "b + a".
L'opération d'addition est associative dans "R".
</p>
<math><mforall>a, b <min> R, a + b = b + a</math>

<h4>3.0.0.3</h4>
<p>
Démontrons que la structure algébrique formée par l'ensemble des réels et de l'opération d'addition est un groupe abélien.
</p>
<math>(R, +)</math>
<p>
Déjà, cette opération est associative sur "R" selon le théorème 3.0.0.1, et elle est commutative selon le théorème 3.0.0.2.
De plus, elle admet un neutre, qui est "0".
</p>
<math><mforall>a, b, c, n <min> R, <mexists>A, B, C, D, E, F, M, N <min> P(Q), a = (A, B), b = (C, D), c = (E, F), n = (M, N)</math>
<math>a + 0 = n = (M, N) <mequi> [<mforall>o <min> A, <mforall>q <min> M, <mforall>r <min> N, q <mgt> o + 0 = q <mand> r <mlt>q] <mequi> a = n</math>
<p>
Finalement, chaque élément possède un inverse donnant le neutre par l'addition.
</p>
<math>a - a = c = (E, F) <mequi> [<mforall>o <min> A, <mforall>e <min> E, <mforall>f <min> F, e <mgt> o - o = 0, f <mlt> e] <mequi> a - a = c = 0</math>
<p>
Donc, la structure algébrique formée par l'ensemble des réels et de l'opération d'addition est bel et bien un groupe abélien.
</p>
<math>(R, +)</math>

<h3 id="real_multiplication">L'opération de multiplication</h3>
<h4>3.0.1.0</h4>
<p>
Démontrons que la multiplication de deux nombres réel non-nul "a" et "b" donne un nombre réel (et donc, que la multiplication est stable dans "R*").
Soit deux nombres réel "a" et "b" :
</p>
<math><mforall>a, b <min> R*, <mexists>A, B, C, D <min> P(Q), a = (A, B), b = (C, D)</math>
<p>
Admettons un objet "g", représentant "a * b".
Nous pouvons l'écrire avec la formule de la multiplication de nombres réels.
</p>
<math>g = a * b <mequi> <mexists> E, F <min> P(Q), g = (E, F), <mforall> f <min> F, <mforall>c <min> A, <mforall>d <min> C, f <mgt> c * d <mand> <mforall> e <min> E, <mforall> f <min> F, e <mlt> f</math>
<p>
Par définition, "E" et "F" sont des sous-parties de "Q", et la relation d'ordre entre leurs éléments prouve que "E" et "F" sont complémentaires.
Donc, "g" est un nombre réel.
</p>

<h4>3.0.1.1</h4>
<p>
Démontrons que l'ordre de multiplication de deux trois nombres réels non-nuls "a", "b" et "c" n'importe pas (et donc, que la multiplication est associative dans "R").
Soit trois nombres réels "a", "b" et "c" :
</p>
<math><mforall>a, b, c <min> R, <mexists>A, B, C, D, E, F <min> P(Q), a = (A, B), b = (C, D), c = (E, F)</math>
<p>
Admettons un objet "j", représentant "(a * b) * c".
Nous pouvons l'écrire avec la formule de la multiplication de nombres réels.
</p>
<math>j = (a * b) * c <mequi> <mexists>G, H <min> P(Q), j = (G, H) <mequi> <mforall> h <min> H, <mforall>c <min> A, <mforall>d <min> C, <mforall> e <min> E, h <mgt> (c * d) * e <mand> <mforall> g <min> G, <mforall> h <min> H, g <mlt> h</math>
<p>
Comme la multiplication est associative dans "Q", on peut modifier l'ordre des multiplications pour définir "j".
</p>
<math>j = (a * b) * c <mequi> <mexists>G, H <min> P(Q), j = (G, H) <mequi> <mforall> h <min> H, <mforall>c <min> A, <mforall>d <min> C, <mforall> e <min> E, h <mgt> (c * d) * e = c + (d + e) <mand> <mforall> g <min> G, <mforall> h <min> H, g <mlt> h <mequi> j = a + (b + c)</math>
<p>
Or, cela consiste à dire que l'on peut aussi modifier l'ordre des multiplications dans "R".
Donc, "(a * b) * c" est égal à "a * (b * c)".
L'opération de multiplication est associative dans "R".
</p>
<math><mforall>a, b, c <min> R, (a * b) * c = a * (b * c)</math>

<h4>3.0.1.2</h4>
<p>
Démontrons que l'ordre de deux opérandes "a" et "b" autour d'une multiplication de nombres réels n'importe pas.
Soit deux nombres réels non-nuls "a" et "b" :
</p>
<math><mforall>a, b <min> R*, <mexists>A, B <min> P(Q), a = (A, B), b = (C, D)</math>
<p>
Admettons un objet "j", représentant "a * b".
Nous pouvons l'écrire avec la formule de la multiplication de nombres réels.
</p>
<math>j = a * b <mequi> <mexists>G, H <min> P(Q), j = (G, H) <mequi> <mforall> h <min> H, <mforall>c <min> A, <mforall>d <min> C, <mforall> e <min> E, h <mgt> a * b <mand> <mforall> g <min> G, <mforall> h <min> H, g <mlt> h</math>
<p>
Comme la multiplication est commutative dans "Q", on peut modifier l'ordre des opérandes pour définir "j".
</p>
<math>j = a * b <mequi> <mexists>G, H <min> P(Q), j = (G, H) <mequi> <mforall> h <min> H, <mforall>c <min> A, <mforall>d <min> C, <mforall> e <min> E, h <mgt> a * b = b* a <mand> <mforall> g <min> G, <mforall> h <min> H, g <mlt> h <mequi> j = b * a</math>
<p>
Or, cela consiste à dire que l'on peut aussi modifier l'ordre des opérandes dans une multiplication dans "R*".
Donc, "a * b" est égal à "b * a".
L'opération de multiplication est associative dans "R".
</p>
<math><mforall>a, b <min> R*, a* b = b * a</math>

<h4>3.0.1.3</h4>
<p>
Démontrons que la structure algébrique formée par l'ensemble des réels non-nuls et de l'opération de multiplication est un groupe abélien.
</p>
<math>(R*, *)</math>
<p>
Déjà, cette opération est associative sur "R*" selon le théorème 3.0.1.1, et elle est commutative selon le théorème 3.0.1.2.
De plus, elle admet un neutre, qui est "1".
</p>
<math><mforall>a, b, c, n <min> R, <mexists>A, B, C, D, E, F, M, N <min> P(Q), a = (A, B), b = (C, D), c = (E, F), n = (M, N)</math>
<math>a * 1 = n = (M, N) <mequi> [<mforall>o <min> A, <mforall>q <min> M, <mforall>r <min> N, q <mgt> o * 1 = q <mand> r <mlt>q] <mequi> a = n</math>
<p>
Finalement, chaque élément possède un inverse donnant le neutre par la multiplication.
</p>
<math>a * <mfrac><mi>1</mi><mi>a</mi></mfrac> = c = (E, F) <mequi> [<mforall>o <min> A, <mforall>e <min> E, <mforall>f <min> F, e <mgt> o / o = 1, f <mlt> e] <mequi> a * <mfrac><mi>1</mi><mi>a</mi></mfrac> = c = 1</math>
<p>
Donc, la structure algébrique formée par l'ensemble des réels non-nuls et de l'opération de multiplication est bel et bien un groupe abélien.
</p>
<math>(R*, *)</math>



<h2>Introduction à l'analyse</h2>

<h3>Les espaces métriques</h3>
<h4>3.1.0.0</h4>
<p>
Démontrons que la valeur absolue de la différence entre deux nombres réel dans l'ensemble des réels forme un espace métrique.
Pour cela, démontrons que cet opérateur sur cet ensemble respecte les propriétés d'une "distance".
</p>
<p>
Démontrons que cet opérateur est symétrique.
Déjà,  :
</p>
<math><mforall>a, b <min> R, a - b = -(-(a - b)) = -(b - a)</math>
<p>
Comme on applique la valeur absolue :
</p>
<math>|a - b| = |-(b - a)| = |b - a|</math>
<p>
Donc, cet opérateur est symétrique.
</p>
<p>
Démontrons que cet opérateur est séparé.
Supposons que "a - b = 0" : nous pouvons additionner "b" des deux côtés.
</p>
<math>|a - b| = 0 <mequi> a = b</math>
<p>
Donc, "a - b = 0" équivaut à dire que "a = b" : cet opérateur est séparé.
</p>
<p>
Finalement, démontrons que cet opérateur respecte l'inégalité triangulaire.
Posons la somme de la distance entre "a" et "b" et de la distance de "b" et "c", que nous nommerons "s".
</p>
<math><mforall> c <min> R, s = |a - b| + |b - c|</math>
<p>
Si "a" est supérieur ou égal à "b", "|a - b| = a - b", alors on a deux cas.
Si "b" est supérieur ou égal à "c", "|b - c| = b - c", et donc :
</p>
<math>a - b + b - c = a - c = |a - c|</math>
<p>
Si "b" est inférieur à "c", "|b - c| = c - b" (qui est positif, et que nous noterons "e"), et donc :
</p>
<math>a - b + c - b = a + c - b - b = a + e - b = a + e - (c - e) = a - c + 2 * e <mgtequal> a - c = [|a - c| si a <mgtequal> c, -|a - c| sinon] <mgtequal> |a - c|</math>
<p>
Tous les cas vérifient cette inégalité.
Maintenant, si "a" est inférieur à "b", "|a - b| = b - a" (qui est positif, et que nous noterons "f"), alors on a deux cas.
Si "b" est supérieur ou égal à "c", "|b - c| = b - c", et donc :
</p>
<math>b - a + b - c = b + b - a - c = b + f - c = f + a + f - c = a - c + 2 * f <mgtequal> a - c = [|a - c| si a <mgtequal> c, -|a - c| sinon] <mgtequal> |a - c|</math>
<p>
Si "b" est inférieur à "c", "|b - c| = c - b", et donc :
</p>
<math>b - a + c - b = -a + c = c - a = |a - c|</math>
<p>
L'inégalité triangulaire est démontrée dans tous les cas : elle est vérifiée dans "R" avec cet opérateur.
Donc, cet opérateur et "R" forme un espace métrique.
</p>

<h3>Les limites de suites usuelles</h3>
<h4>3.1.1.0</h4>
<p>
Montrons que la suite où l'on divise "1" par l'indice de l'élément actuel de la suite tend vers "0".
Appelons cette suite "u(n)".
</p>
<math>u(n) = <mfrac><mi>1</mi><mi>n</mi></mfrac></math>
<p>
Posons "x", représentant un nombre réel positif quelconque (différent de 0), supérieur à 0.
</p>
<math><mforall> x <min> R*+</math>
<p>
Supposons l'existence d'un "m" tel que la valeur absolue de "u(m)" soit inférieur à "x".
Cela équivaut à dire que "1" divisé par "m" est inférieur à "x".
</p>
<math><mforall> m <min> N, abs(u(m)) <mlt> a <mequi> <mfrac><mi>1</mi><mi>n</mi></mfrac> <mlt> x</math>
<p>
Comme "m" et "x" sont supérieur à "0", alors "1" divisé par "m" est aussi supérieur à "0", et l'on peut obtenir une équivalence en appliquant la fonction inverse des deux côtés.
</p>
<math><mfrac><mi>1</mi><mi>n</mi></mfrac> <mlt> x <mequi> m <mlt> <mfrac><mi>1</mi><mi>x</mi></mfrac></math>
<p>
Donc, "m" doit être supérieur à "1" divisé par "x".
Donc, pour tout nombre réel "x" supérieur à 0, il existe une valeur "b" tel que, pour toute valeur "m" de la suite supérieure à "b", "u(m)" est plus proche de 0 que "x".
Donc, par définition de la limite, la suite où l'on divise "1" par l'indice de l'élément actuel de la suite tend vers "0".
</p>
<math>u(n) = <mfrac><mrow><mi>1</mi></mrow><mrow><mi>n</mi></mrow></mfrac>, lim u(n) = 0</math>

<h4>3.1.1.1</h4>
<p>
Montrons que la suite représentant le produit de "n" par un nombre "a" avec une puissance positive tend vers "+ l'infini" si "a" est positif, et "- l'infini" si "a" est négatif.
Appelons cette suite "u(n)".
</p>
<math><mforall> a <min> R, <mforall> e <min> N*, u(n) = a * <mi>n</mi><msup>e</msup></math>
<p>
Commençons par le cas où "a" est supérieur à "0".
Posons "x", représentant un nombre réel positif quelconque (différent de 0), supérieur à 0.
</p>
<math><mforall> x <min> R*+</math>
<p>
Supposons l'existence d'un "m" tel que la valeur absolue de "u(m)" soit supérieur à "x".
Cela équivaut à dire que le produit de "n" par un nombre "a" avec une puissance positive est inférieur à "x".
</p>
<math><mforall> m <min> N, abs(u(m)) <mgt> x <mequi> a * <mi>m</mi><msup>e</msup> <mgt> x</math>
<p>
Comme "a" est supérieur à "0", on peut diviser par "a" des deux côtés.
</p>
<math>a * <mi>m</mi><msup>e</msup> <mgt> x <mequi> <mi>m</mi><msup>e</msup> <mgt> <mfrac><mi>x</mi><mi>a</mi></mfrac></math>
<p>
Donc, "m" doit être supérieur à la racine de "x" divisé par "a" base "e".
Donc, pour tout nombre réel "x" supérieur à 0, il existe une valeur "b" tel que, pour toute valeur "m" de la suite supérieure à "b", "u(m)" est supérieur à "x".
Donc, par définition de la limite, la suite représentant le produit de "n" par un nombre "a" positif avec une puissance positive tend vers "+ l'infini".
</p>
<p>
Finissons par le cas où "a" est inférieur à "0".
Posons "x", représentant un nombre réel négatif quelconque (différent de 0), supérieur à 0.
</p>
<math><mforall> x <min> R*-</math>
<p>
Supposons l'existence d'un "m" tel que la valeur absolue de "u(m)" soit inférieur à "x".
Cela équivaut à dire que le produit de "n" par un nombre "a" négatif avec une puissance positive est inférieur à "x".
</p>
<math><mforall> m <min> N, abs(u(m)) <mgt> x <mequi> a * <mi>m</mi><msup>e</msup> <mlt> x</math>
<p>
Comme "a" est inférieur à "0", on peut diviser par "a" des deux côtés, mais on doit changer le signe de la relation d'ordre.
</p>
<math>a * <mi>m</mi><msup>e</msup> <mlt> x <mequi> <mi>m</mi><msup>e</msup> <mgt> <mfrac><mi>x</mi><mi>a</mi></mfrac></math>
<p>
Donc, "m" doit être supérieur à la racine de "x" divisé par "a" (qui est un nombre positif) base "e".
Donc, pour tout nombre réel "x" inférieur à 0, il existe une valeur "b" tel que, pour toute valeur "m" de la suite supérieure à "b", "u(m)" est inférieur à "x".
Donc, par définition de la limite, la suite représentant le produit de "n" par un nombre "a" négatif avec une puissance positive tend vers "- l'infini".
</p>

<h3>Les limites de fonctions usuelles</h3>
<h4>3.1.2.0</h4>
<p>
Montrons que la limite en "+ l'infini" d'une fonction réelle est la même que la limite de la suite qui lui est associée.
Commençons par le cas où la limite est une limite finie.
Posons une fonction "f" admettant une limite "v" en "+ l'infini".
</p>
<math>f : A <mto> B, a <mto> f(b)</math></br>
<math>lim f(a) = v</math>
<p>
Exprimons la définition de la limite.
</p>
<math><mforall> x <mgt> 0, <mexists> a <min> R, <mforall> b <min> R, [b <mgt> a <mimp> |f(b) - v| <mlt> x]</math>
<p>
Or, on sait que la partie entière de "a" auquelle on ajoute "1" est strictement supérieure à "a".
</p>
<math><mforall> a <min> R, ent(a) + 1 <mgt> a</math>
<p>
Appelons "c" la valeur partie entière de "a" auquelle on ajoute "1".
Notre définition de la limite implique donc une définition plus précise :
</p>
<math><mforall> x <mgt> 0, <mexists> c <min> N, <mforall> b <min> R, [b <mgt> c <mimp> |f(b) - v| <mlt> x]</math>
<p>
Nous savons aussi que tout nombre réel "b" supérieur à "a" permet de faire fonctionner la limite.
Donc, c'est la même chose pour tout nombre entier naturel "d" supérieur à "a", car "N" est une partie de "R".
</p>
<math><mforall> x <mgt> 0, <mexists> c <min> N, <mforall> d <min> N, [d <mgt> c <mimp> |f(d) - v| <mlt> x]</math>
<p>
Appelons "u" la suite associé à la fonction "f".
Cette suite représente toutes les valeurs de "f" quand l'antécédent est un nombre entier naturel.
Or, c'est le cas dans "f(d)" : il est parfaitement égal à "u(d)".
</p>
<math><mforall> x <mgt> 0, <mexists> c <min> N, <mforall> d <min> N, [d <mgt> c <mimp> |u(d) - v| <mlt> x]</math>
<p>
Or, cela est exactement la définition d'une limite de suite vers "v" en "+ l'infini".
Donc, "u" tend bel et bien vers "v".
Donc, la suite associée à fonction réelle tendant vers "v" en "+ l'infini" tend aussi vers "v" en "+ l'infini".
</p>
<p>
Maintenant, traitons le cas où la limite est "+ l'infini".
Posons une fonction "f" admettant une limite "+ l'infini" en "+ l'infini".
</p>
<math>f : A <mto> B, a <mto> f(b)</math></br>
<math>lim f(a) = + <minf></math>
<p>
Exprimons la définition de la limite.
</p>
<math><mforall> x <mgt> 0, <mexists> a <min> R, <mforall> b <min> R, [b <mgt> a <mimp> f(b) <mgt> x]</math>
<p>
Or, on sait que la partie entière de "a" auquelle on ajoute "1" est strictement supérieure à "a".
</p>
<math><mforall> a <min> R, ent(a) + 1 <mgt> a</math>
<p>
Appelons "c" la valeur partie entière de "a" auquelle on ajoute "1".
Notre définition de la limite implique donc une définition plus précise :
</p>
<math><mforall> x <mgt> 0, <mexists> c <min> N, <mforall> b <min> R, [b <mgt> c <mimp> f(b) <mgt> x]</math>
<p>
Nous savons aussi que tout nombre réel "b" supérieur à "a" permet de faire fonctionner la limite.
Donc, c'est la même chose pour tout nombre entier naturel "d" supérieur à "a", car "N" est une partie de "R".
</p>
<math><mforall> x <mgt> 0, <mexists> c <min> N, <mforall> d <min> R, [d <mgt> c <mimp> f(d) <mgt> x]</math>
<p>
Appelons "u" la suite associé à la fonction "f".
Cette suite représente toutes les valeurs de "f" quand l'antécédent est un nombre entier naturel.
Or, c'est le cas dans "f(d)" : il est parfaitement égal à "u(d)".
</p>
<math><mforall> x <mgt> 0, <mexists> c <min> N, <mforall> d <min> R, [d <mgt> c <mimp> u(d) <mgt> x]</math>
<p>
Or, cela est exactement la définition d'une limite de suite vers "+ l'infini" en "+ l'infini".
Donc, "u" tend bel et bien vers "+ l'infini".
Donc, la suite associée à fonction réelle tendant vers "+ l'infini" en "+ l'infini" tend aussi vers "+ l'infini" en "+ l'infini".
</p>
<p>
Finalement, traitons le cas où la limite est "- l'infini".
Posons une fonction "f" admettant une limite "- l'infini" en "+ l'infini".
</p>
<math>f : A <mto> B, a <mto> f(b)</math></br>
<math>lim f(a) = - <minf></math>
<p>
Exprimons la définition de la limite.
</p>
<math><mforall> x <mgt> 0, <mexists> a <min> R, <mforall> b <min> R, [b <mgt> a <mimp> f(b) <mlt> -x]</math>
<p>
Or, on sait que la partie entière de "a" auquelle on ajoute "1" est strictement supérieure à "a".
</p>
<math><mforall> a <min> R, ent(a) + 1 <mgt> a</math>
<p>
Appelons "c" la valeur partie entière de "a" auquelle on ajoute "1".
Notre définition de la limite implique donc une définition plus précise :
</p>
<math><mforall> x <mgt> 0, <mexists> c <min> N, <mforall> b <min> R, [b <mgt> c <mimp> f(b) <mlt> -x]</math>
<p>
Nous savons aussi que tout nombre réel "b" supérieur à "a" permet de faire fonctionner la limite.
Donc, c'est la même chose pour tout nombre entier naturel "d" supérieur à "a", car "N" est une partie de "R".
</p>
<math><mforall> x <mgt> 0, <mexists> c <min> N, <mforall> d <min> R, [d <mgt> c <mimp> f(d) <mlt> -x]</math>
<p>
Appelons "u" la suite associé à la fonction "f".
Cette suite représente toutes les valeurs de "f" quand l'antécédent est un nombre entier naturel.
Or, c'est le cas dans "f(d)" : il est parfaitement égal à "u(d)".
</p>
<math><mforall> x <mgt> 0, <mexists> c <min> N, <mforall> d <min> R, [d <mgt> c <mimp> u(d) <mlt> -x]</math>
<p>
Or, cela est exactement la définition d'une limite de suite vers "- l'infini" en "+ l'infini".
Donc, "u" tend bel et bien vers "- l'infini".
Donc, la suite associée à fonction réelle tendant vers "- l'infini" en "+ l'infini" tend aussi vers "- l'infini" en "+ l'infini".
</p>
<p>
Tous les cas sont vérifiés : la limite en "+ l'infini" d'une fonction réelle est la même que la limite de la suite qui lui est associée.
</p>

<h3>L'arithmétique des limites</h3>
<h4>3.1.3.0</h4>
<p>
Démontrons que dans une fonction "f" représentant la somme d'autres fonctions, la limite de "f" en une valeur représente le produit des limites en cette valeur des fonctions qui constituent "f".
La démonstration de ce résultat n'est pas si complexe que ça, mais demande une disjonction de cas précise.
Posons deux fonctions "g" et "h" définies de "R" vers "R", admettant une limite à droite en une valeur finie "a" vers respectivement "v" et "w" finis.
</p>
<math><mforall> x <min> R*+, <mexists> b <min> R*+, {<mforall> c <min> R, c <min> ]a, a + b[ <mimp> |g(c) - v| <mlt> x}</math></br>
<math><mforall> x <min> R*+, <mexists> d <min> R*+, {<mforall> e <min> R, e <min> ]a, a + d[ <mimp> |h(c) - w| <mlt> x}</math>
<p>
Procédons à un petit tour de passe-passe : prenons la valeur minimale entre "b" et "d".
Si "b" est inférieur à "d", alors "b" est un "e", et si "d" est inférieur à "b", alors "d" est un "c" : travailler avec la valeur minimale suffit pour définir "c" et "e".
Nous nommerons "r" la valeur minimale entre "b" et "d", et nous pouvons simplifier nos limites de "g" et "h".
</p>
<math><mforall> x <min> R*+, <mexists> r <min> R*+, {<mforall> s <min> R, s <min> ]a, a + r[ <mimp> |g(s) - v| <mlt> x <mand> |h(s) - w| <mlt> x}</math></br>
<p>
Posons maintenant la fonction "f" égale à "g * h".
On peut bidouiller notre formule précédente pour obtenir une limite de "f".
</p>
<math><mforall> x <min> R*+, <mexists> r <min> R*+, {<mforall> s <min> R, s <min> ]a, a + r[ <mimp> 0 <mlt> |g(s) - v| <mlt> x <mand> 0 <mlt> |h(s) - w| <mlt> x <mimp> 0 <mlt> |g(s) - v| * |h(s) - w| <mlt> x * x} <mand> {<munder><mi>lim</mi><mrow><mi>x</mi><mto><mi>a</mi></mrow></munder>((g(s) * -w) + (h(s) * -v)) = 0}</math></br>
<math>g(s) - v <mgt> 0 <mand> h(s) - w <mgt> 0 <mimp> 0 <mlt> (g(s) - v) * (h(s) - w) <mlt> x * x <mequi> 0 <mlt> g(s) * h(s) - (v * w) + (g(s) * -w) + (h(s) * -v) <mlt> x * x <mequi> f(s) - (v * w) <mlt> x * x <mimp> 0 <mlt> |f(s) - (v * w)| <mlt> x * x</math></br>
<math>g(s) - v <mgt> 0 <mand> h(s) - w <mlt> 0 <mimp> 0 <mlt> (g(s) - v) * (-h(s) + w) <mlt> x * x <mequi> 0 <mlt> (g(s) - v) * (h(s) - w) * -1 <mlt> x * x <mequi> 0 <mlt> (f(s) - (v * w)) * -1 <mlt> x * x <mimp> |f(s) - (v * w)| <mlt> x * x</math></br>
<math>g(s) - v <mlt> 0 <mand> h(s) - w <mgt> 0 <mimp> 0 <mlt> (-g(s) + v) * (h(s) - w) <mlt> x * x <mequi> 0 <mlt> (g(s) - v) * (h(s) - w) * -1 <mlt> x * x <mequi> 0 <mlt> (f(s) - (v * w)) * -1 <mlt> x * x <mimp> |f(s) - (v * w)| <mlt> x * x</math></br>
<math>g(s) - v <mlt> 0 <mand> h(s) - w <mlt> 0 <mimp> 0 <mlt> (-g(s) + v) * (-h(s) + w) <mlt> x * x <mequi> 0 <mlt> (g(s) - v) * (h(s) - w) <mlt> x * x <mimp> 0 <mlt> |f(s) - (v * w)| <mlt> x * x</math></br>
<p>
Dans tous les cas, on retrouve la limite de "f" : la limite de "f" en "a" est le produit des limites de "g" et "h" en "a".
C'est l'exact même chose pour les limites à gauches.
</p>
<math><mforall> x <min> R*+, <mexists> b <min> R*+, {<mforall> c <min> R, c <min> ]a - b, a[ <mimp> |g(c) - v| <mlt> x}</math></br>
<math><mforall> x <min> R*+, <mexists> d <min> R*+, {<mforall> e <min> R, e <min> ]a - d, a[ <mimp> |h(c) - w| <mlt> x}</math>
<math><mforall> x <min> R*+, <mexists> r <min> R*+, {<mforall> s <min> R, s <min> ]a - r, a[ <mimp> 0 <mlt> |g(s) - v| <mlt> x <mand> 0 <mlt> |h(s) - w| <mlt> x <mimp> 0 <mlt> |g(s) - v| * |h(s) - w| <mlt> x * x} <mand> {<munder><mi>lim</mi><mrow><mi>x</mi><mto><mi>a</mi></mrow></munder>((g(s) * -w) + (h(s) * -v)) = 0}</math></br>
<math>g(s) - v <mgt> 0 <mand> h(s) - w <mgt> 0 <mimp> 0 <mlt> (g(s) - v) * (h(s) - w) <mlt> x * x <mequi> 0 <mlt> g(s) * h(s) - (v * w) + (g(s) * -w) + (h(s) * -v) <mlt> x * x <mequi> f(s) - (v * w) <mlt> x * x <mimp> 0 <mlt> |f(s) - (v * w)| <mlt> x * x</math></br>
<math>g(s) - v <mgt> 0 <mand> h(s) - w <mlt> 0 <mimp> 0 <mlt> (g(s) - v) * (-h(s) + w) <mlt> x * x <mequi> 0 <mlt> (g(s) - v) * (h(s) - w) * -1 <mlt> x * x <mequi> 0 <mlt> (f(s) - (v * w)) * -1 <mlt> x * x <mimp> |f(s) - (v * w)| <mlt> x * x</math></br>
<math>g(s) - v <mlt> 0 <mand> h(s) - w <mgt> 0 <mimp> 0 <mlt> (-g(s) + v) * (h(s) - w) <mlt> x * x <mequi> 0 <mlt> (g(s) - v) * (h(s) - w) * -1 <mlt> x * x <mequi> 0 <mlt> (f(s) - (v * w)) * -1 <mlt> x * x <mimp> |f(s) - (v * w)| <mlt> x * x</math></br>
<math>g(s) - v <mlt> 0 <mand> h(s) - w <mlt> 0 <mimp> 0 <mlt> (-g(s) + v) * (-h(s) + w) <mlt> x * x <mequi> 0 <mlt> (g(s) - v) * (h(s) - w) <mlt> x * x <mimp> 0 <mlt> |f(s) - (v * w)| <mlt> x * x</math></br>
<p>
On peut étendre ces résultats pour des limites en "+ l'infini" et "- l'infini".
Dans ce cas, notre "r" devient la plus grande valeur entre "b" et "d".
On reste dans le cas où "v" et "w" sont des valeurs finies.
Commençons par la limite en "+ l'infini".
</p>
<math><mforall> x <min> R*+, <mexists> b <min> R*+, {<mforall> c <min> R, c <mgt> b <mimp> |g(c) - v| <mlt> x}</math></br>
<math><mforall> x <min> R*+, <mexists> d <min> R*+, {<mforall> e <min> R, e <mgt> d <mimp> |h(e) - w| <mlt> x}</math></br>
<math><mforall> x <min> R*+, <mexists> r <min> R*+, {<mforall> s <min> R, s <mgt> r <mimp> 0 <mlt> |g(s) - v| <mlt> x <mand> 0 <mlt> |h(s) - w| <mlt> x <mimp> 0 <mlt> |g(s) - v| + |h(s) - w| <mlt> 2 * x}</math></br>
<math>g(s) - v <mgt> 0 <mand> h(s) - w <mgt> 0 <mimp> 0 <mlt> (g(s) - v) * (h(s) - w) <mlt> x * x <mequi> 0 <mlt> g(s) * h(s) - (v * w) + (g(s) * -w) + (h(s) * -v) <mlt> x * x <mequi> f(s) - (v * w) <mlt> x * x <mimp> 0 <mlt> |f(s) - (v * w)| <mlt> x * x</math></br>
<math>g(s) - v <mgt> 0 <mand> h(s) - w <mlt> 0 <mimp> 0 <mlt> (g(s) - v) * (-h(s) + w) <mlt> x * x <mequi> 0 <mlt> (g(s) - v) * (h(s) - w) * -1 <mlt> x * x <mequi> 0 <mlt> (f(s) - (v * w)) * -1 <mlt> x * x <mimp> |f(s) - (v * w)| <mlt> x * x</math></br>
<math>g(s) - v <mlt> 0 <mand> h(s) - w <mgt> 0 <mimp> 0 <mlt> (-g(s) + v) * (h(s) - w) <mlt> x * x <mequi> 0 <mlt> (g(s) - v) * (h(s) - w) * -1 <mlt> x * x <mequi> 0 <mlt> (f(s) - (v * w)) * -1 <mlt> x * x <mimp> |f(s) - (v * w)| <mlt> x * x</math></br>
<math>g(s) - v <mlt> 0 <mand> h(s) - w <mlt> 0 <mimp> 0 <mlt> (-g(s) + v) * (-h(s) + w) <mlt> x * x <mequi> 0 <mlt> (g(s) - v) * (h(s) - w) <mlt> x * x <mimp> 0 <mlt> |f(s) - (v * w)| <mlt> x * x</math></br>
<p>
C'est la même chose pour les limites en "- l'infini".
</p>
<math><mforall> x <min> R*+, <mexists> b <min> R*+, {<mforall> c <min> R, c <mlt> -b <mimp> |g(c) - v| <mlt> x}</math></br>
<math><mforall> x <min> R*+, <mexists> d <min> R*+, {<mforall> e <min> R, e <mlt> -d <mimp> |h(e) - w| <mlt> x}</math></br>
<math><mforall> x <min> R*+, <mexists> r <min> R*+, {<mforall> s <min> R, s <mlt> -r <mimp> 0 <mlt> |g(s) - v| <mlt> x <mand> 0 <mlt> |h(s) - w| <mlt> x <mimp> 0 <mlt> |g(s) - v| + |h(s) - w| <mlt> 2 * x}</math></br>
<math>g(s) - v <mgt> 0 <mand> h(s) - w <mgt> 0 <mimp> 0 <mlt> (g(s) - v) * (h(s) - w) <mlt> x * x <mequi> 0 <mlt> g(s) * h(s) - (v * w) + (g(s) * -w) + (h(s) * -v) <mlt> x * x <mequi> f(s) - (v * w) <mlt> x * x <mimp> 0 <mlt> |f(s) - (v * w)| <mlt> x * x</math></br>
<math>g(s) - v <mgt> 0 <mand> h(s) - w <mlt> 0 <mimp> 0 <mlt> (g(s) - v) * (-h(s) + w) <mlt> x * x <mequi> 0 <mlt> (g(s) - v) * (h(s) - w) * -1 <mlt> x * x <mequi> 0 <mlt> (f(s) - (v * w)) * -1 <mlt> x * x <mimp> |f(s) - (v * w)| <mlt> x * x</math></br>
<math>g(s) - v <mlt> 0 <mand> h(s) - w <mgt> 0 <mimp> 0 <mlt> (-g(s) + v) * (h(s) - w) <mlt> x * x <mequi> 0 <mlt> (g(s) - v) * (h(s) - w) * -1 <mlt> x * x <mequi> 0 <mlt> (f(s) - (v * w)) * -1 <mlt> x * x <mimp> |f(s) - (v * w)| <mlt> x * x</math></br>
<math>g(s) - v <mlt> 0 <mand> h(s) - w <mlt> 0 <mimp> 0 <mlt> (-g(s) + v) * (-h(s) + w) <mlt> x * x <mequi> 0 <mlt> (g(s) - v) * (h(s) - w) <mlt> x * x <mimp> 0 <mlt> |f(s) - (v * w)| <mlt> x * x</math></br>
<p>
Dans le cas de limites en "+ l'infini" et en "- l'infini", il faut faire plus attention à bien respecter la règle des signes.
Commençons dans le cas de limites en une valeur finie.
Si nous avons deux limites à droites en "+ l'infini".
</p>
<math><mforall> x <min> R*+, <mexists> b <min> R*+, {<mforall> c <min> R, c <min> ]a, a + b[ <mimp> g(c) <mgt> x}</math></br>
<math><mforall> x <min> R*+, <mexists> d <min> R*+, {<mforall> e <min> R, e <min> ]a, a + d[ <mimp> h(e) <mgt> x}</math>
<math><mforall> x <min> R*+, <mexists> r <min> R*+, {<mforall> s <min> R, s <min> ]a, a + r[ <mimp> 0 <mlt> g(s) <mgt> x <mand> 0 <mlt> h(s) <mgt> x <mimp> g(s) * h(s) <mgt> x * x <mimp> f(x) <mgt> x * x}</math></br>
<p>
Et à gauche.
</p>
<math><mforall> x <min> R*+, <mexists> b <min> R*+, {<mforall> c <min> R, c <min> ]a - b, a[ <mimp> g(c) <mgt> x}</math></br>
<math><mforall> x <min> R*+, <mexists> d <min> R*+, {<mforall> e <min> R, e <min> ]a - d, a[ <mimp> h(e) <mgt> x}</math>
<math><mforall> x <min> R*+, <mexists> r <min> R*+, {<mforall> s <min> R, s <min> ]a - r, a[ <mimp> 0 <mlt> g(s) <mgt> x <mand> 0 <mlt> h(s) <mgt> x <mimp> g(s) * h(s) <mgt> x * x <mimp> f(x) <mgt> x * x}</math></br>
<p>
Maintenant, si nous avons deux limites à droites en "- l'infini".
</p>
<math><mforall> x <min> R*+, <mexists> b <min> R*+, {<mforall> c <min> R, c <min> ]a, a + b[ <mimp> g(c) <mlt> -x}</math></br>
<math><mforall> x <min> R*+, <mexists> d <min> R*+, {<mforall> e <min> R, e <min> ]a, a + d[ <mimp> h(e) <mlt> -x}</math>

<p>

</p>
<math><mforall> x <min> R*+, <mexists> b <min> R*+, {<mforall> c <min> R, c <min> ]a - b, a[ <mimp> g(c) <mlt> -x}</math></br>

<p>
Maintenant, si nous avons une limite à droite en "+ l'infini" et une en "- l'infini".
</p>
<math><mforall> x <min> R*+, <mexists> b <min> R*+, {<mforall> c <min> R, c <min> ]a, a + b[ <mimp> g(c) <mgt> x}</math></br>

<math><mforall> x <min> R*+, <mexists> r <min> R*+, {<mforall> s <min> R, s <min> ]a, a + r[ <mimp> g(s) <mgt> x <mand> h(s) <mlt> -x <mimp> g(s) * h(s) <mlt> -x * x <mimp> f(x) <mlt> -x * x}</math></br>

<math><mforall> x <min> R*+, <mexists> b <min> R*+, {<mforall> c <min> R, c <min> ]a - b, a[ <mimp> g(c) <mgt> x}</math></br>

<math><mforall> x <min> R*+, <mexists> b <min> R*+, {<mforall> c <min> R, c <min> ]a - b, a[ <mimp> g(c) <mlt> -x}</math></br>
<math><mforall> x <min> R*+, <mexists> d <min> R*+, {<mforall> e <min> R, e <min> ]a - d, a[ <mimp> h(e) <mgt> x}</math>
<math><mforall> x <min> R*+, <mexists> r <min> R*+, {<mforall> s <min> R, s <min> ]a - r, a[ <mimp> g(s) <mgt> x <mand> h(s) <mlt> -x <mimp> g(s) * h(s) <mlt> -x * x <mimp> f(x) <mlt> -x * x}</math></br>
<p>
C'est la même chose en "+ l'infini" (dans tous les cas).
</p>
